# 网文创作技巧手册

> 更新时间：2026-02-20
> 来源：Tavily 搜索整合

---

## 一、基础功：写前准备

### 1.1 大纲是骨架
- **必须列大纲**（除非短篇）
- 大纲包含：主线剧情、支线设定、人物成长线、关键转折点
- 建议用「章纲」形式，每章列出核心事件和目标

### 1.2 资料搜集
- 写玄幻 → 了解人物等级体系、技能树设定
- 写都市 → 了解行业规则、职场细节
- 写历史 → 考据时代背景、风俗礼仪
- **原则**：写到某领域时，资料要足够支撑细节

### 1.3 世界观构建
- 完整的世界观设定（包括地理、势力、货币、等级制度）
- 规则体系要自圆其说，不能前后矛盾
- 世界观服务于剧情，不是堆砌设定

## 二、核心技巧：细节刻画

### 2.1 环境描写
| ❌ 差 | ✅ 好 |
|------|------|
| 房间很乱 | 书籍像秋天的落叶散落一地，衣物从椅子上垂落，宛如疲倦旅人的叹息 |
| 他走进森林 | 脚下的落叶沙沙作响，空气中弥漫着腐叶与泥土的气息，阳光从枝叶缝隙漏下 |

**技巧**：调动五感（视觉、听觉、嗅觉、味觉、触觉）

### 2.2 人物内心
| ❌ 差 | ✅ 好 |
|------|------|
| 他很紧张 | 他不自觉地摩挲手指，嘴唇微微发白，呼吸变得急促 |
| 她很伤心 | 眼眶泛红，却倔强地昂着头，指甲深深掐入掌心 |

**技巧**：用动作、神态、身体反应替代直接陈述情绪

### 2.3 对话刻画
- **语言反映性格**：果断的人说话简短有力，温柔的人话语细腻柔和
- **留白艺术**：让读者想象未言之语背后的含义
- **对话推动情节**：每句对白要有信息量或冲突

### 2.4 配角塑造
- 每个配角至少有一次**独立行动**（非陪衬主角）
- 有自己的动机和立场
- 通过配角视角丰富主线剧情

## 三、爽点设计（网文核心）

### 3.1 爽点类型
| 类型 | 示例 |
|------|------|
| **打脸** | 前期轻视主角的人后期后悔莫及 |
| **逆袭** | 废物主角突然崛起，震惊全场 |
| **收获** | 获得宝物、功法、势力 |
| **情感** | 甜甜的恋爱、兄弟情义 |
| **解谜** | 揭开伏笔、真相大白 |

### 3.2 节奏把控
- **黄金三章**：开篇要抓住读者，前3章决定留存率
- **飞卢风**：快节奏切入，冲突密集，签约后适当扩展
- **起点风**：世界观宏大，升级体系清晰，长期积累
- **章节钩子**：每章结尾留悬念，引导继续阅读

### 3.3 节奏失衡警告
- ❌ 节奏拖沓 → 减少冗余描述，加快剧情推进
- ❌ 节奏太快 → 增加人物互动、情感铺垫
- ❌ 爽点密集疲劳 → 适当插入低谷情节

## 四、避坑指南

### 4.1 常见问题
| 问题 | 原因 | 解决 |
|------|------|------|
| 流水账 | 大纲/素材不够详细 | 先建立详细素材库 |
| 数字出错 | 写完不检查 | 建立检查清单 |
| 质量下降 | 没有质检标准 | 建立自检清单 |
| 人设崩塌 | 写着写着跑偏 | 随时对照人物档案 |
| 节奏失控 | 压缩太狠/扩展太水 | 详细大纲设计 |

### 4.2 自检清单（每章必做）
- [ ] 场景具体吗？（说"在哪"不说"什么圈子"）
- [ ] 有画面感吗？（闭眼能想象吗？）
- [ ] 数字感知了吗？（大数字换成可感描述）
- [ ] 反派存在吗？（有明确对手吗？）
- [ ] 情感共鸣了吗？（触发基础情绪了吗？）

### 4.3 错误分级
| 级别 | 定义 | 处理 |
|------|------|------|
| P0 致命 | 逻辑矛盾、人设崩塌、核心伏笔断裂 | 必须重写 |
| P1 严重 | 数字错误、常识硬伤、时间线混乱 | 24小时内修复 |
| P2 一般 | 文笔平淡、节奏拖沓、配角抢戏 | 迭代优化 |
| P3 建议 | 可改可不改、锦上添花 | 记录待办 |

## 五、风格统一

### 5.1 三大统一
| 统一项 | 内容 |
|--------|------|
| **叙事节奏** | 长短句搭配、段落长度、场景切换频率 |
| **语言习惯** | 用词偏好、修辞风格、角色口头禅 |
| **情感基调** | 幽默/严肃、轻快/沉重、热血/温情 |

### 5.2 AI 辅助（保持风格）
- 用 AI 降重时，**风格一致性高于一切**
- 保留原文的叙事节奏和语言习惯
- 检测：对比修改前后的情感基调是否一致

## 六、创作 SOP（推荐流程）

```
S1 素材库 → S2 大纲设计 → S3 分镜脚本 → S4 滚动执行 → S5 质量检查 → S6 发布归档
```

| 阶段 | 产出 | 质检 |
|------|------|------|
| S1 素材库 | 核心概念卡、素材库 | 素材完整度 |
| S2 大纲 | 大纲.md、人物档案.md、卷纲 | 层级清晰、人物动线明确 |
| S3 分镜 | 分镜脚本 + 矛盾设计 | 五项自检通过 |
| S4 正文 | 正文 + 章节摘要 | 符合分镜、上下文连贯 |
| S5 质检 | 检查报告 | P0 致命错误清零 |
| S6 发布 | 可发布稿件 | 可直接发布 |
