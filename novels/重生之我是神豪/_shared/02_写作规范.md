# 写作规范 v1.0

## 一、核心原则

### 爽文公式
```
爽感 = 细节堆砌 × 情绪放大 × 反差冲击 × 周围人反应
```

### 单章结构（4000字）
| 阶段 | 篇幅 | 内容 |
|------|------|------|
| 铺垫 | 30% | 场景描写、被打压 |
| 逆袭 | 25% | 主角反击 |
| 打脸 | 25% | 对方震惊反应 |
| 收获 | 15% | 得到好处 |
| 钩子 | 5% | 下一章悬念 |

---

## 二、身临感技巧

### 1. 五感描写法
```md
# ❌ 原文：店员态度很差
# ✅ 改进：
她从头到脚扫了我一遍。
洗得发白的T恤。
淘宝79元的裤子。
穿了两年的回力鞋。
"买得起吗？"她小声嘀咕。
```

### 2. NPC侧面描写法（重点！！）
```md
# ✅ 正确写法：增加旁观者反应
# 场景：主角买手机

整个苹果店瞬间安静了一秒。

"我操——"
"俩学生买了两台iPhone？"
"三万！眼睛都不眨？"

旁边一对情侣正在挑手机。
女的拽了拽男朋友。
"你看看别人！"
"你再看看你！"
```

### 3. 情绪放大法
```md
# ❌ 原文：周明轩很开心
# ✅ 改进：
周明轩心脏狂跳！
血液直冲天灵盖！
老子TMD终于有钱了！！！
```

### 3. 四阶段被打脸法
```md
阶段1：轻视（眼神、语气）
阶段2：愣住（动作僵硬）
阶段3：震惊（手指发抖）
阶段4：谄媚（180度态度转变）
```

### 4. 细节堆砌法
```md
# ❌ 原文：买了辆很贵的车
# ✅ 改进：
幻影EWB，官方指导价920万。
选配：星光顶、后排娱乐、冰箱...
落地超过1000万。
V12发动机，6.7T马力。
百公里加速5.3秒。
```

## 三、爽点密度

### 每章要求
| 爽点类型 | 数量 | 说明 |
|----------|------|------|
| **核心大爽点** | 1个 | 本章主要打脸/逆袭 |
| **小爽点** | 3-4个 | 穿插在各段落 |
| **情绪高潮** | 2-3个 | 感叹号、内心独白 |

### 爽点分布
```
800字 → 1个小爽点
2000字 → 核心大爽点
4000字 → 4-5个爽点
```

## 四、系统使用规范

### 系统通知格式
```md
【系统通知】
【今日收益已到账：144,000元】
【当前账户余额：XXX元】
【VIP等级：青铜VIP】
```

### 禁止出现
- ❌ 银行短信（改为系统APP通知）
- ❌ 银行卡扣款短信（改为系统扣款）
- ❌ 分钟级收入短信（改为每日汇总）
- ❌ 税务/反洗钱相关

---

## 五、财富来源处理

### 对外解释
- **统一口径**："家族海外信托"、"祖父遗产"
- **态度**：简单带过，不详细解释
- **被追问**：岔开话题或反问"很重要吗？"

### 避免合理性质疑
- 不出现税务调查
- 不出现反洗钱冻结
- 不出现亲戚借钱
- 不出现仇富报警

---

## 六、禁止项清单

### 情节禁止
| 类别 | 禁止内容 |
|------|---------|
| 系统 | 每分钟短信/分钟级收入 |
| 金融 | 银行卡风控/冻结/调查 |
| 法律 | 税务问题/洗钱嫌疑 |
| 人物 | 主角OOC/女主绿茶/反派洗白 |
| 节奏 | 超过1500字无爽点 |

### 用词规范
| 避免 | 替换 |
|------|------|
| 短信通知 | 系统APP推送 |
| 银行卡 | 系统账户 |
| 分钟收入 | 日收入 |
| 现金 | 系统虚拟货币 |

## 七、衔接规范

### 每章必须包含
- [ ] 本章爽点（4-5个）
- [ ] 系统提示（1-2次）
- [ ] 被打脸方反应（4阶段）
- [ ] 下一章钩子

### 章节衔接
- 结尾必须埋钩子
- 下一章开头必须提上一章
- 人物状态要一致

---

## 八、检查清单（写完后自检）

### 爽点检查
- [ ] 是否有4-5个爽点？
- [ ] 是否使用了五感描写？
- [ ] 被打脸反应是否分4阶段？

### 系统检查
- [ ] 系统通知是否正确格式？
- [ ] 是否有财富来源质疑？

### 防毒点检查（重点！！！）
- [ ] **数字是否正确计算**？（3000×12=36000，不是3万）
- [ ] **金额是否符合身份**？（年薪10万说"穷"是毒点）
- [ ] **反应是否合理**？（"微微惊讶"是毒点，应该是"震惊"）
- [ ] **是否有逻辑错误**？（"3万没有零头"是毒点）
- [ ] **时间线是否一致**？

### 衔接检查
- [ ] 结尾是否埋了钩子？
- [ ] 字数是否在3500-4500字？
