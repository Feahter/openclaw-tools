# Attention System Skills
# 注意力分析捕获正向循环系统
# 基于元认知与第一性原理设计

## 核心架构

### 第一性原理定义
- **注意力 = 神经资源的定向分配** (40比特/1100万比特)
- **注意力 = 时间的前置货币** (认知⇄时间⇄信任)
- **注意力 = 信息熵的降低承诺** (确定性供给或新信息供给)

### 元认知双系统
- **系统A (捕获系统)**: 设计信息穿透机制
- **系统B (反捕获系统)**: 避免零和博弈与虚假紧迫感
- **关键**: 系统A输出成为系统B输入，形成自我校准

## Skill 1: 趋势感知雷达 (Trend Sensing Radar)

### 功能
多源异构信息扫描，识别注意力重新分配的交接点

### 触发条件
- 用户需要分析新领域/市场/人群
- 制定季度/年度内容战略
- 验证现有假设是否过时

### 执行步骤

#### 1.1 环境扫描 (Sensing)
```
输入: 目标领域关键词
输出: 四维度信号矩阵

维度1: 技术成熟度
- 指标: 工具门槛是否降至"一句话指令"
- 信号源: Product Hunt, GitHub Trending, 技术博客
- 当前热点: AI生成工具普及、低代码自动化成熟

维度2: 政策红利期  
- 指标: 补贴/准入/税收窗口
- 信号源: 国务院公报、地方产业扶持政策
- 当前热点: 银发经济、新能源、乡村振兴

维度3: 代际行为迁移
- 指标: "不得不"的新习惯形成
- 信号源: 视频号/小红书/抖音趋势报告
- 当前热点: 私域闭环、AI工具依赖、情绪消费

维度4: 供应链重构
- 指标: 去中介化或再中介化节点
- 信号源: 跨境电商报告、本地生活数据
- 当前热点: 本对本电商、即时配送网络
```

#### 1.2 认知缝隙识别
```
操作: 在目标平台搜索"怎么用XX" / "XX教程"
判断标准:
- 高频提问 + 低质回答 = 高价值缝隙
- 情绪词密集 (焦虑/困惑/求推荐) = 紧迫需求
- 官方文档缺失或过时 = 机会窗口

输出: 3-5个可验证的"认知缝隙"假设
```

#### 1.3 反共识校准
```
触发器: 当某领域所有人说"必须做XX"
操作:
1. 标记该建议为"可能已过度"
2. 寻找相邻领域的成熟模式 (认知时差)
3. 输出: 3个"逆向机会"假设
```

### 输出格式
```json
{
  "trend_analysis": {
    "domain": "目标领域",
    "signal_matrix": {
      "tech_maturity": {"score": 1-10, "evidence": []},
      "policy_window": {"score": 1-10, "evidence": []},
      "behavior_shift": {"score": 1-10, "evidence": []},
      "supply_chain": {"score": 1-10, "evidence": []}
    },
    "cognitive_gaps": [
      {"gap": "缝隙描述", "urgency": "高/中/低", "platform": "主阵地"}
    ],
    "contrarian_opportunities": []
  }
}
```

## Skill 2: 认知钩子编码器 (Cognitive Hook Encoder)

### 功能
将原始信号转化为可穿透注意力过滤机制的信息单元

### 触发条件
- 已有内容主题，需要设计传播形式
- 内容表现不佳，需要重构信息结构
- 建立标准化内容模板库

### 执行步骤

#### 2.1 五元素拆解 (不可约简要素)
```
任何注意力交换必须包含:

1. 触发器 (Trigger)
   - 定义: 打破自动化的意外性
   - 检验: 0.3秒内制造认知失调
   - 类型: 数字冲突/身份挑战/反常识事实/悬念缺口

2. 模式 (Pattern)  
   - 定义: 可预测的结构降低处理成本
   - 检验: 用户不思考就能"感觉"到规律
   - 类型: 三段式/对比式/递进式/循环式

3. 缺口 (Gap)
   - 定义: 已知与想知之间的张力
   - 检验: 创造"必须 closure"的开放循环
   - 类型: 知识缺口/能力缺口/认同缺口

4. 共鸣 (Resonance)
   - 定义: 自我相关性的瞬间确认
   - 检验: 用户立即想到"这就是我"
   - 类型: 身份标签/场景还原/情绪命名

5. 出口 (Exit)
   - 定义: 明确的下一步行动
   - 检验: 低摩擦的注意力承接容器
   - 类型: 关注/私信/加群/保存/转发
```

#### 2.2 情绪波长匹配
```
分析目标人群的当前情绪基线:
- 焦虑型: 提供确定性 ("3步解决XX")
- 好奇型: 制造缺口 ("90%的人不知道...")
- 愤怒型: 身份确认 ("我们XX人早就...")
- 认同型: 价值共鸣 ("终于有人说明白了...")

禁忌: 在焦虑人群中制造更多焦虑 (不可持续)
```

#### 2.3 记忆锚定设计
```
操作: 将抽象概念绑定到具体感官经验
- 视觉锚: 强对比色/特定形状/场景画面
- 听觉锚: 押韵/节奏/重复句式
- 动作锚: 可模仿的手势/操作流程

检验: 用户24小时后能否无提示回忆核心信息
```

### 输出格式
```json
{
  "hook_design": {
    "content_theme": "原始主题",
    "five_elements": {
      "trigger": {"type": "", "execution": "", "test": ""},
      "pattern": {"type": "", "structure": "", "predictability": ""},
      "gap": {"type": "", "tension_level": "", "closure_promise": ""},
      "resonance": {"identity_tag": "", "scenario": "", "emotion_label": ""},
      "exit": {"action": "", "friction_level": "", "next_container": ""}
    },
    "emotion_match": "目标情绪基线",
    "memory_anchor": {"sensory_channel": "", "binding_target": ""}
  }
}
```

## Skill 3: 最小可验证单元 (MVE - Minimum Viable Engagement)

### 功能
快速测试注意力捕获假设，区分噪音与信号

### 触发条件
- 新领域验证 (不确定用户是否买单)
- 资源有限 (无法大规模生产)
- 快速迭代需求 (需要周级反馈)

### 执行步骤

#### 3.1 MVE定义标准
```
不是MVP (产品), 是MVE (注意力单元)

形式选择:
- 短视频: 15-60秒, 单点信息, 完播率>40%
- 图文: 3-5屏, 一个钩子+一个出口, 保存率>5%
- 直播切片: 3分钟精华, 互动率>3%
- 问答: 单问题深度回答, 追问率>10%

核心标准: 用户能立即行动 (关注/私信/加群)
```

#### 3.2 神经反馈指标监测
```
区分两类反馈:

噪音反馈 (情绪反应):
- 点赞/评论情绪词 ("厉害了"/"支持")
- 高流量低转化 (泛娱乐内容常见)
- 不可累积, 单次消耗

信号反馈 (行为改变):
- 保存/转发/私信/付费动作
- 完播率>互动率>粉丝增长
- 可累积, 信任沉淀

决策规则: 信号反馈权重 = 噪音反馈 × 3
```

#### 3.3 假设-测试循环
```
每个MVE必须包含:
1. 明确假设: "如果我们用X方式呈现Y内容,Z人群会A行动"
2. 可证伪标准: "如果24小时内B指标未达到C数值,假设不成立"
3. 变量控制: 每次只改一个元素 (标题/封面/开头/结尾)
4. 失败熔断: 连续3次未达标准,回溯到编码层重新设计

记录模板:
- 假设内容
- 测试日期/平台
- 关键指标数据
- 结论: 验证/证伪/需调整
- 模式提取 (如验证成功)
```

### 输出格式
```json
{
  "mve_test": {
    "hypothesis": "",
    "falsifiability": {"metric": "", "threshold": "", "timeframe": ""},
    "execution": {"platform": "", "format": "", "content_snapshot": ""},
    "results": {
      "noise_signals": {"views": 0, "likes": 0, "comments": 0},
      "true_signals": {"saves": 0, "shares": 0, "privates": 0, "conversion": 0}
    },
    "conclusion": "验证/证伪/调整",
    "pattern_extraction": "",
    "next_action": ""
  }
}
```

## Skill 4: 双环学习系统 (Double-Loop Learning)

### 功能
从单环策略调整升级到双环目标质疑

### 触发条件
- 某个内容表现异常好/差 (偏离常态)
- 连续成功/失败后的模式固化风险
- 季度战略复盘

### 执行步骤

#### 4.1 单环学习 (策略层)
```
操作: 优化执行方式
- A/B测试标题/封面/发布时间
- 调整内容长度/节奏/密度
- 优化出口引导话术

边界: 不改变目标本身
适用: 已知有效模式的规模化
```

#### 4.2 双环学习 (目标层)
```
触发问题:
1. 这是系统噪音 (运气) 还是结构信号 (模式)?
   - 检验: 能否在相似条件下复现?

2. 如果复制这个"成功",会强化什么认知框架?
   - 检验: 该框架的边界条件是什么?

3. 我们为什么要做这类内容?
   - 检验: 目标是否与长期信任积累一致?

操作: 
- 每月选一天"逆向日",强制输出与主流观点相悖内容
- 建立"反指标"清单: 当所有人都说"必须做XX",标记为过度
- 定期(季度)回顾"第一性假设",淘汰过时框架
```

#### 4.3 认知框架升级
```
成功模式提取:
- 抽象出可迁移的结构 (而非复制具体形式)
- 标注情境变量 (在什么条件下有效)
- 建立"如果...那么..."的规则库

失败模式熔断:
- 记录"死亡信号" (什么特征预示必然失败)
- 设定自动止损点 (如连续3次MVE失败即暂停)
- 分析"近失" (差一点成功的情况,往往包含关键信息)
```

### 输出格式
```json
{
  "learning_loop": {
    "trigger_event": "异常表现描述",
    "single_loop": {
      "adjustments": [],
      "test_results": []
    },
    "double_loop": {
      "noise_vs_signal": "判断依据",
      "framework_reinforced": "",
      "boundary_conditions": [],
      "goal_alignment": "一致/冲突/需调整"
    },
    "framework_update": {
      "new_rules": [],
      "deprecated_rules": [],
      "uncertainty_areas": []
    }
  }
}
```

## Skill 5: 信任飞轮构建 (Trust Flywheel)

### 功能
将一次性注意力转化为可累积的信任资产

### 触发条件
- 已有初始流量，需要沉淀私域
- 设计长期用户生命周期
- 建立护城河 (抗平台算法变化)

### 执行步骤

#### 5.1 飞轮结构
```
内容 (获取注意力) 
  → 私域 (沉淀信任) [微信/社群/邮件列表]
    → 服务 (变现) [课程/咨询/产品]
      → 案例 (反哺内容) [用户成功故事]
        → 强化触发器敏感度 (循环)

关键: 每个环节必须降低下一环节的摩擦
```

#### 5.2 信任账户管理
```
每笔注意力交易记账:
- 存款: 用户获得真实认知收益 (新知/洞察/连接)
- 取款: 用户支付时间/信任/金钱
- 规则: 存款必须大于取款,且可累积

检验标准:
- 用户注意力投入后,认知收益是否真实?
- 循环迭代后,内容质量上升(更深/更准)还是下降(更俗/更躁)?
- 系统是否具备"优雅退出"机制?

伦理红线: 不剥削认知弱点,不制造虚假紧迫感
```

#### 5.3 反脆弱设计
```
从波动中获益的机制:
- 平台算法变化 → 私域资产保值
- 内容形式过时 → 认知框架可迁移
- 竞争加剧 → 信任壁垒提高切换成本

操作:
- 内容分发: 多平台,但私域为最终容器
- 能力沉淀: 提取可迁移模式,非绑定特定工具
- 收入结构: 信任变现占比逐年提升
```

### 输出格式
```json
{
  "trust_flywheel": {
    "current_stage": "内容/私域/服务/案例",
    "friction_points": [],
    "trust_account": {
      "deposits": ["认知收益类型"],
      "withdrawals": ["用户付出类型"],
      "balance_trend": "上升/持平/下降"
    },
    "antifragility": {
      "platform_risk": "高/中/低",
      "format_risk": "高/中/低", 
      "competition_risk": "高/中/低",
      "mitigation": []
    }
  }
}
```

## Skill 6: 认知卸载工具箱 (Cognitive Offloading)

### 功能
外部化工作记忆，释放元认知监控能力

### 触发条件
- 信息过载，无法聚焦
- 决策疲劳，质量下降
- 需要建立长期可观测性

### 执行步骤

#### 6.1 感知层卸载
```
工具: RSS聚合器 + 自动化监控
- Feedly/Inoreader订阅高质量源
- 设定"每周扫描"时段,非实时响应
- 关键词警报(品牌/竞品/趋势词)

输出: 每周"信号摘要",标记异常值供人工处理
```

#### 6.2 编码层卸载
```
工具: "钩子库"管理系统
- 按五元素分类存储高互动案例
- 标签系统: 情绪类型/平台/形式/表现数据
- 定期(月度)结构复盘,非形式复制

模板:
- 原始案例链接
- 五元素拆解
- 情境变量(平台/时间/受众)
- 可迁移结构抽象
```

#### 6.3 反馈层卸载
```
工具: 数据仪表板
- 自动抓取多平台核心指标
- 区分噪音vs信号可视化
- 异常值自动标记

决策日志模板:
- 决策日期/内容
- 第一性假设
- 预期结果
- 实际结果
- 偏差分析
- 模式更新
```

#### 6.4 演化层卸载
```
工具: 规则库版本控制
- 成功/失败模式结构化存储
- 规则变更历史追踪
- 季度框架升级记录

输出: 个人"注意力语法手册"——内部语言系统
```

### 输出格式
```json
{
  "offloading_system": {
    "sensing_tools": [],
    "encoding_library": {"categories": [], "last_review": ""},
    "feedback_dashboard": {"metrics": [], "automation_level": ""},
    "evolution_repository": {"rules_count": 0, "version": ""},
    "manual_status": "草稿/完善中/成熟"
  }
}
```

## 整合执行路线图

### Phase 1: 基础建设 (4-6周)
- 每日: 15分钟"注意力审计" (记录自己被什么捕获)
- 每周: 运行1个最小循环 (感知→编码→反馈)
- 产出: 个人"注意力语法手册"初稿

### Phase 2: 模式识别 (8-12周)  
- 建立信号库与钩子库
- 运行假设-测试循环
- 产出: 3-5个验证过的认知钩子模板

### Phase 3: 系统自治 (持续)
- 自动化感知与反馈
- 模板化生产
- 周期性升级 (季度框架回顾)

## 使用说明

1. **按需调用**: 根据当前任务选择对应Skill,不必全链路执行
2. **顺序依赖**: Skill 3依赖Skill 2输出,Skill 5依赖前期流量基础
3. **元认知监控**: 全程保持系统B(反捕获)在线,定期伦理自检
4. **失败熔断**: 连续3次MVE失败即回溯,避免沉没成本陷阱
