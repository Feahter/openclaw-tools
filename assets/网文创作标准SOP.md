# 网文创作标准SOP

> **版本**: v1.0  
> **更新日期**: 2026-02-18  
> **状态**: 生效中

---

## 核心理念

融合「工程化六阶段」与「种子→萌芽→枝干→枝叶四层架构」，形成统一的创作流程。

**核心原则**：先确定，再执行。素材→大纲→分镜→创作→质检→归档

---

## 一、四层架构（方法论底层）

| 层级 | 产出 | 触发时机 | 必做产出 |
|------|------|---------|---------|
| **L1 种子层** | 核心概念卡 | 新小说创建 | 核心概念卡.md |
| **L2 萌芽层** | 大纲 + 人物档案 + 卷纲 | 新卷开始 | 大纲.md + 人物档案.md + 卷纲 |
| **L3 枝干层** | 分镜脚本 + 章节规划 | 每章创作前 | 分镜脚本 + 矛盾设计 |
| **L4 枝叶层** | 正文 + 章节摘要 | 每章创作时 | 正文 + 章节摘要 |

### 执行原则

1. **严格按层级执行** - 必须完成当前层级才能进入下一层级
2. **每层必做产出** - 不能跳过任何必做项
3. **执行前检查** - 开始创作前先确认当前层级和产出

---

## 二、六阶段标准流程

| 阶段 | 名称 | 产出 | 质量门槛 |
|------|------|------|---------|
| **S1 素材库** | 收集详细素材 | 核心概念卡、素材库 | — |
| **S2 大纲设计** | 详细设定 | 大纲.md、人物档案.md、卷纲 | 层级清晰、人物动线明确 |
| **S3 分镜脚本** | 每章详细规划 | 分镜脚本 + 矛盾设计 | 五项自检通过 |
| **S3.5 技术规格** | 章节技术指标 | 规格表 | 字段完整、指标明确 |
| **S4 滚动执行** | 写→验证→调整 | 正文 + 章节摘要 | 符合分镜、上下文连贯、完读率自估 |
| **S5 质量检查** | 数字逻辑、常识、NPC反应 | 检查报告 | P0致命错误清零 |
| **S6 发布归档** | 整理校对、格式统一 | 可发布稿件 | 可直接发布 |

> **阶段耦合说明**：S3→S3.5→S4 为串行链路，S3.5 依赖 S3 产出，若 S3 修改需同步更新 S3.5

---

## 三、分镜脚本五项自检（强制）

**写完后必须自检，不合格则退回重写：**

| 检验项 | 要求 | 不合格示例 | 合格示例 |
|--------|------|------------|----------|
| **场景具体** | 说"在哪"不说"什么圈子" | "进入上流圈子" | "私人会所VIP包间" |
| **画面感** | 闭眼能想象画面 | "成为焦点" | "记者的长枪短炮对准主角" |
| **数字感知** | 大数字换算成可感描述 | "14万亿" | "每人一百万需要14亿人" |
| **反派存在** | 每章有明确对手 | "主角成功了" | "秦昊脸色铁青" |
| **情感共鸣** | 每章至少触发1种基础情绪（恐惧/愤怒/喜悦/惊讶/厌恶/悲伤），主角行为符合马斯洛需求层次 | "捐50万亿" | "当年嫌他穷的前女友后悔了" |

### 自检流程

```
1. 这一章的核心画面是什么？用一句话描述出来
2. 没有文字，读者能"看到"什么？
3. 这个数字/场景跟我有什么关系？
4. 对手是谁？他的反应是什么？
5. 普通人会怎么想这个场景？
```

**质量门槛**：任何一项不满足 → 退回重写

## 四、S3.5 技术规格（章节技术指标）

| 字段 | 说明 | 示例 |
|------|------|------|
| **信息密度** | 高/中/低，控制节奏 | 高密度=每段有事件推进 |
| **情绪曲线** | 起→承→转→合，标注峰值点 | "愤怒→爆发→沉默" |
| **钩子位置** | 第几段埋入下一章钩子 | 第5段结尾 |
| **伏笔回收** | 关联前文章节号 | 第3章埋，第12章收 |
| **对话占比** | 目标30%-50%，超出预警 | 40% |

## 五、人物弧光检查清单（每3章填写）

### 主角
- [ ] 能力/认知是否有变化？（从__到__）
- [ ] 人际关系是否有推进？（新增/破裂/深化）
- [ ] 是否面临更难的选择？（代价递增）

### 核心配角
- [ ] 每人至少1次独立行动（非陪衬主角）
- [ ] 每人至少1次与主角的冲突/合作

### 反派
- [ ] 威胁等级是否升级？（从__到__）
- [ ] 是否有"合理动机"展示？（避免脸谱化）

## 六、阶段质检清单

| 阶段 | 质检项 |
|------|--------|
| S1 素材库 | 素材完整度、核心概念清晰度 |
| S2 大纲 | 情节逻辑、人物弧光、节奏安排 |
| S3 分镜 | 五项自检全部通过 |
| S4 正文 | 符合分镜、无错别字、对话自然、黄金三章出口质检、每3章回溯检查 |
| S5 质检 | 数字准确、常识无误、NPC反应合理、P0错误清零 |

---

## 七、错误分级标准

| 级别 | 定义 | 处理方式 |
|------|------|---------|
| **P0 致命** | 逻辑矛盾、人设崩塌、核心伏笔断裂 | 必须重写 |
| **P1 严重** | 数字错误、常识硬伤、时间线混乱 | 24小时内修复 |
| **P2 一般** | 文笔平淡、节奏拖沓、配角抢戏 | 迭代优化 |
| **P3 建议** | 可改可不改、锦上添花 | 记录待办 |

---

## 八、常见错误与解决

| 错误 | 原因 | 解决 |
|------|------|------|
| 偷懒写流水账 | 素材库/大纲不详细 | 先建立详细素材 |
| 数字出错 | 写完不检查 | 建立检查清单 |
| 质量下降 | 没有质检标准 | 建立自检清单 |
| 节奏失控 | 压缩太狠 | 详细大纲设计 |
| 层级混乱 | 没按流程执行 | 严格按S1-S6执行 |

---

## 九、回溯检查机制

### 定期触发
每写完3章主动回溯检查人物弧光和上下文连贯性

### 事件触发（立即回溯）
- 发现人物行为/性格与设定不符
- 出现逻辑矛盾或伏笔丢失
- 读者反馈或自查发现明显问题
- 任何P0/P1级错误被发现

> **原则**：定期检查是预防，事件触发是纠偏，两者互补

---

## 十、执行约定

1. **严格按序** - 必须完成当前阶段才能进入下一阶段
2. **每阶段必做** - 不能跳过任何阶段的必做产出
3. **质检贯穿** - 不是最后才质检，每章写完就自检
4. **层级检查** - 创作前先确认当前层级和必做产出

## 附录：矛盾设计模板

### 核心矛盾公式

```
主角目标 + 阻碍力量 = 冲突
```

### 矛盾类型

| 类型 | 示例 |
|------|------|
| **生存矛盾** | 资源匮乏、生命威胁 |
| **利益矛盾** | 权力争夺、财富分配 |
| **情感矛盾** | 爱恨交织、信任危机 |
| **理念矛盾** | 价值观冲突、信念对立 |
| **身份矛盾** | 阶层差距、身份对立 |

### 章节矛盾设计检查

- [ ] 本章核心冲突是什么？
- [ ] 主角的主动选择是什么？
- [ ] 反派的阻碍动作是什么？
- [ ] 冲突结果如何影响后续？
- [ ] 是否有情绪钩子埋入？
