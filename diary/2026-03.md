## 03-07

### Skill 创建 SOP 总结

用户询问创建 Skills 的 SOP是什么。总结如下：

#### 完整流程

```
请求创建 Skill
      ↓
skill-from-masters 触发（方法论研究）
      ↓
   任务缩窄（5层框架：领域→受众→格式→目标→约束）
      ↓
  方法论研究（专家/书籍/最佳实践）
      ↓
  整合专家框架
      ↓
skill-creator 执行创建（SKILL.md编写）
      ↓
   SKILL.md + scripts/ + references/
      ↓
   安装部署（clawdhub install / 手动）
```

#### 核心 Skills

| Skill | 功能 |
|-------|------|
| skill-from-masters | 基于专家方法论创建高质量 Skills |
| skill-creator | 执行创建的具体指导 |

#### 关键原则

1. **任务定义要狭窄、具体** - "写邮件"❌ → "给 CTO 写 B2B 冷启动邮件"✅
2. **模型选择** - 非技术性任务用 Opus 级别模型
3. **方法论研究** - 搜索+验证，直到得出清晰可靠的结论
4. **简洁优先** - 只添加 Claude 不知道的内容
