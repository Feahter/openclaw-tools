### 快速决策表

| 问题 | 推荐算法 | 复杂度 | 代码位置 |
|------|---------|--------|---------|
| 连通集合合并 | 并查集 Union-Find | O(α(n)) | `## 并查集` |

| 单点更新+前缀和 | 树状数组 BIT | O(log n) | `## 树状数组` |
| 静态区间和 | 前缀和 | O(1) | `## 前缀和+差分` |
| 第 K 大（一次性） | 快速选择 QuickSelect | O(n) | `## 快速选择` |
| Top-K（动态） | 堆 heap | O(n log k) | `## 堆` |
| 最近最少使用 | LRU（链表+哈希） | O(1) | `## LRU` |
| 图社区检测 | Leiden / Louvain | O(n log n) | `## Leiden` |
| 回文子串 | Manacher | O(n) | `## Manacher` |

## ANALYZE 模式

### 问题类型 → 推荐算法

| 问题 | 推荐算法 | 复杂度 |
|------|---------|--------|
| Top-K、最大的K个 | 堆 or 快速选择 | O(n log k) / O(n) |
| 去重、是否存在 | 哈希表 | O(1) |
| 大范围存在（可误判） | 布隆过滤器 | O(k) 空间 |
| 区间和 | BIT or 线段树 | O(log n) |
| 图连通、集合合并 | 并查集 | O(α(n)) |
| 回文 | Manacher | O(n) |
| 有序查找 | 二分 | O(log n) |
| 最近最少使用 | LRU | O(1) |
| 递归爆栈 | 显式栈 / Morris | O(n) |
| ⚠️ 字符串匹配 | KMP（待补充）| O(n+m) |
| ⚠️ 最短路 | Dijkstra（待补充）| O(E log V) |

### ANALYZE 输出格式

```
## 分析结论

**问题类型**：[分类]
**推荐算法**：[名字]
**复杂度**：时间 / 空间
**不适合场景**：[如果有]
**选择原因**：[一句话]

→ 说"给我实现"进入 LOOKUP
```

## RESEARCH 模式 — 7步内嵌SOP

### 典型对话

```
用户：研究一下Tarjan算法

你：了解，按7步走。

## 1. 算法定性
| 维度 | 分析 |
|------|------|
| 本质 | 图的强连通分量算法 |
| 核心思想 | DFS + low-link 值判断 |
| 时间复杂度 | O(V+E) |
| 空间复杂度 | O(V) |

## 2. 与经典算法对比
| 经典算法 | 共同点 | Tarjan 改进 |
| Kosaraju | 找SCC | 只需一次DFS，无需反向图 |

...（继续7步）
```

### 7步模板

**S1 定性**：本质 + 核心思想 + 复杂度 + 解决什么问题

**S2 对比**：找家族，理解改进了什么

**S3 公式**：核心数学公式 + 变量说明

**S4 代码**：分阶段解读 + 关键设计决策

**S5 优化**：时间/空间/并行化方向

**S6 迁移**：应用场景 + 注意事项

**S7 实现 + 归档**：
```python
# 基础版（50-150行）
```
```python
# 优化版（150-400行）
```
→ 追加到 `references/algorithms.md`
→ 决策表追加一行
