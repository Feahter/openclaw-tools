# 算法深度研究 SOP — 7步法

> 完整研究流程见 skill-algo-master RESEARCH 模式

---

## Step 1: 算法定性

先搞清楚算法"是什么"，用一句话定位：

```
[算法名] = [本质类型] + [核心思想]
```

**定性维度表：**

| 维度 | 要回答的问题 |
|------|-------------|
| 本质 | 属于哪类算法？（搜索/排序/图/优化/ML...） |
| 核心思想 | 一句话说清楚它在干什么 |
| 搜索类型 | 贪心/动态规划/回溯/随机/进化... |
| 时间复杂度 | O(?) |
| 空间复杂度 | O(?) |
| 解决什么问题 | 前置算法的什么缺陷？ |

---

## Step 2: 与经典算法对比

找到"家族"，理解它改进了什么：

| 经典算法 | 共同点 | 本算法改进点 |
|---------|-------|------------|
| [算法A] | [共同点] | [改进点] |
| [算法B] | [共同点] | [改进点] |

**搜索方向**：
- `[算法名] vs [相关算法]`
- arxiv.org / Semantic Scholar / Google Scholar

---

## Step 3: 核心公式/数据结构

**公式提取：**
```
核心公式 = 算法的灵魂
```

**关键数据结构：**
```
[数据结构名] — [用途] — [为什么选它]
```

---

## Step 4: 核心代码解析

**关键函数识别**：
```bash
grep -r "def \|function \|fn \|func " <core_file> | head -20
grep -n "while\|for\|loop\|iterate" <core_file> | head -10
```

**代码解析模板**：

```
### [函数名]（[阶段名]）

[一句话说这个函数干什么]

[关键代码片段]

关键：[最重要的一行，解释为什么这样做]
```

---

## Step 5: 优化方向分析

| 当前实现 | 可优化方案 | 适用场景 |
|---------|-----------|---------|
| [现有策略] | [改进方案] | [什么时候用] |

**常见优化维度：**
- 时间：O(n²) → O(n log n)
- 空间：减少内存占用
- 并行化：串行 → 多线程/分布式
- 精度/速度权衡：精确 → 近似算法
