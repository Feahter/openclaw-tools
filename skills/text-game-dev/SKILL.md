# 离线版文字游戏设计与开发

> 本 Skill 专注**垂直领域**（文字游戏）。通用游戏设计请使用 `game-designer-toolkit`；Web 游戏开发请使用 `game-engine`；设计思维请使用 `game-design-philosophy`

## 核心框架

## 核心框架

### 1. 核心玩法要素

| 要素 | 设计要点 |
|------|---------|
| **文本呈现** | 字体、字号、颜色、行间距精心设计 |
| **选择系统** | 选择应有明确后果反馈，避免"虚假选择" |
| **进度管理** | 存档、读档、自动存档 |
| **回放功能** | 回顾已阅读文本和已做选择 |

### 2. 玩家能力设计顺序

按首次体验顺序介绍：从基础操作（点击推进、显示/隐藏UI）→ 复杂功能（快速存档、历史记录、设置）

## 叙事结构与分支设计

### 分支叙事四原则

1. **有意义的选择** - 每个选择对故事有实际影响
2. **后果可见性** - 即时或延迟显现，但必须让玩家感受重要性
3. **角色一致性** - 分支角色行为符合既定性格
4. **主题锚定** - 所有分支围绕核心主题

### 分支复杂度管理

- **汇聚点设计**：关键节点设置汇聚点，让分支重新汇合
- **模块化场景**：设计可适应多种分支的核心场景

## 引擎选择指南

| 引擎 | 难度 | 费用 | 适用场景 |
|------|------|------|---------|
| **Ren'Py** | 中等 | 免费开源 | 功能强大，社区活跃，需Python |
| **TyranoBuilder** | 简单 | 免费/付费 | 可视化编辑，无需编程 |
| **VN Maker** | 简单 | 付费 | 可视化操作，素材库丰富 |
| **Twine** | 简单 | 免费开源 | 纯文字交互小说，专注分支叙事 |
| **GameCreator** | 简单 | 免费/付费 | 国产引擎，中文支持好 |

**选择建议**：初学者用 TyranoBuilder，进阶用 Ren'Py，纯文字用 Twine

## 美术资源设计

### 角色立绘
- 透明背景（PNG带Alpha）
- 表情差分：多种表情变化
- 风格统一
- 分辨率：PC端建议1920x1080+

### 背景图
- 手绘/照片处理/3D渲染
- 支持格式：JPG、PNG、WEBP

### 推荐工具
- Clip Studio Paint：专业插画
- Photoshop：图像处理
- Stable Diffusion：AI批量生成
- Blender：3D场景渲染

## 音效音乐设计

### 环境音效
- 自然环境：风声、雨声、鸟鸣
- 建筑环境：脚步声、门开关声
- 动态变化：根据时间/天气调整

### 背景音乐
- 根据剧情氛围切换
- 日常→轻快，紧张→节奏感强，悲伤→低沉

### 技术要点
- 循环播放：无缝循环
- 音量控制：独立调节BGM/音效
- 淡入淡出：切换时平滑过渡
- 格式：OGG或MP3

## UI/UX设计原则

- 对话框清晰易读
- 字体清晰易读

- 分辨率设置

- 自动播放速度

## 多结局设计

### 结局类型

| 类型 | 描述 |
|------|------|
| 好结局 | 正确选择后的理想结局 |
| 坏结局 | 错误选择后的结局，具警示作用 |
| 真结局 | 特定条件解锁的隐藏结局 |
| 彩蛋结局 | 趣味性结局，提供惊喜 |

### 触发机制

- **关键选择累积**：记录关键选择，根据倾向决定结局
- **隐藏条件**：好感度、收集物品等

### 结局展示
- 专门的结局CG和音乐
- 结局回顾功能
- 二周目继承部分进度
