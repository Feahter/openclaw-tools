## 色彩系统 {#color}

### 原则

- **3 色规则**：1 个主色 + 1 个强调色 + 中性灰阶
- 背景只用纯白（#FFFFFF）或深色（#1A1A2E / #0D1117）
- 强调色仅用于需要吸引注意力的元素
- 色彩对比度须满足 WCAG AA 标准（4.5:1 以上）

### 推荐配色方案

| 场景 | 背景 | 主色 | 强调色 |
|------|------|------|--------|
| 技术分享 | #0D1117 | #C9D1D9 | #58A6FF |
| 商务汇报 | #FFFFFF | #1F2937 | #2563EB |
| 产品发布 | #000000 | #F5F5F5 | #FF6B35 |
| 培训教学 | #FAFAFA | #374151 | #10B981 |
| 创意提案 | #1A1A2E | #E2E8F0 | #F472B6 |

## 字体排版 {#typography}

### 原则（Edward Tufte）

- 单一字体家族，仅用粗细区分层级
- 行高 1.4-1.6（正文），1.1-1.2（标题）
- 每行不超过 40 个中文字符 / 75 个英文字符

### 字号层级

| 层级 | 用途 | 现场演讲 | 异步阅读 |
|------|------|----------|----------|
| H1 | 页面标题 | 48-64px | 36-48px |
| H2 | 分组标题 | 36-48px | 28-36px |
| Body | 正文/要点 | 24-32px | 18-24px |
| Caption | 说明/来源 | 18-20px | 14-16px |

### 推荐字体

| 语言 | 首选 | 备选 |
|------|------|------|
| 英文 | Inter | SF Pro, Helvetica Neue |
| 中文 | 思源黑体 | PingFang SC, Microsoft YaHei |
| 代码 | JetBrains Mono | Fira Code, SF Mono |

## 布局与留白 {#layout}

### 原则（Garr Reynolds）

- **留白不是浪费** — 留白引导注意力
- 内容占比控制在 40-50%
- 统一使用 8px 基础网格
- 边距最小 48px（现场演讲建议 64px）

### 布局模式

**标题+视觉锚（默认模式）**
```
┌─────────────────────────┐
│                         │
│  标题 = 核心观点        │
│                         │
│  ┌───────────────────┐  │
│  │                   │  │
│  │   视觉锚点        │  │
│  │   （图/表/数字）   │  │
│  │                   │  │
│  └───────────────────┘  │
│                         │
│  • 支撑1  • 支撑2      │
│                         │
└─────────────────────────┘
```

**左文右图（对比/解释模式）**
```
┌─────────────────────────┐
│  标题 = 核心观点        │
│                         │
│  文字区    │  视觉区    │
│  • 要点1   │  ┌──────┐  │
│  • 要点2   │  │ 图表 │  │
│  • 要点3   │  └──────┘  │
│                         │
└─────────────────────────┘
```

**全屏视觉（冲击力模式）**
```
┌─────────────────────────┐
│                         │
│    ┌─────────────────┐  │
│    │                 │  │
│    │   全屏图/数据   │  │
│    │                 │  │
│    └─────────────────┘  │
│                         │
│         标题            │
└─────────────────────────┘
```

**大数字（数据冲击模式）**
```
┌─────────────────────────┐
│                         │
│                         │
│        10x              │
│     部署频率提升         │
│                         │
│   从 1次/周 到 10次/天   │
│                         │
└─────────────────────────┘
```

## 数据可视化 {#dataviz}

### Edward Tufte 核心原则

1. **数据墨水比最大化** — 删除所有不承载数据的视觉元素
2. **消除图表垃圾** — 无 3D 效果、无装饰性网格、无多余图例
3. **小倍数法** — 用相同格式的小图并排对比，优于一张复杂图

### 图表选择

| 目的 | 推荐图表 | 避免 |
|------|----------|------|
| 趋势变化 | 折线图 | 3D 柱状图 |
| 比例构成 | 堆叠条形图 | 饼图（>5 项时） |
| 数量对比 | 水平条形图 | 雷达图 |
| 相关性 | 散点图 | 气泡图（难读） |
| 流程/关系 | 桑基图/流程图 | 复杂网络图 |

### 数据标注

- 直接在数据点上标注数值，不依赖图例
- 高亮关键数据点，弱化其余
- 标题说明结论（"收入增长 3 倍"），非描述（"收入趋势"）

## 动画规范 {#animation}

### 原则

- 动画服务于理解，不是装饰
- 仅使用：渐入（fade-in）、平移（slide）、缩放（scale）
- 时长 300-500ms，缓动函数 ease-out
- 禁止：旋转、弹跳、闪烁、3D 翻转

### 使用场景

| 场景 | 推荐动画 | 目的 |
|------|----------|------|
| 逐步揭示 | fade-in + slide-up | 控制信息节奏 |
| 对比展示 | 左右 slide-in | 并列对比 |
| 数据增长 | 数值计数动画 | 强调变化幅度 |
| 流程演示 | 逐步高亮 | 引导注意力 |
