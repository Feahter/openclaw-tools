## SCQA 模型（Barbara Minto） {#scqa}

> 来源：《金字塔原理》

**适用场景**：问题解决型 — 有明确痛点，需推动方案落地。

**结构**：

```
S - Situation（共识背景）
    听众已经知道并认同的事实。建立共同起点。
    技巧：用"我们都知道..."或数据事实开场。

C - Complication（冲突/变化）
    打破背景的新变量。制造紧张感。
    技巧：用"但是..."、"然而..."、"直到..."转折。

Q - Question（核心疑问）
    从冲突中自然浮现的问题。观众心中已在问。
    技巧：问题应该是听众最想知道答案的那个。

A - Answer（解决方案）
    你的核心论点 + 分论点（MECE 分组）。
    技巧：先给结论，再展开论据。
```

**页面映射示例**（15 分钟演讲）：

```
[封面]     1 页
[S] 背景   1-2 页（快速建立共识，不要拖）
[C] 冲突   1-2 页（制造"啊这确实是个问题"的感觉）
[Q] 问题   0-1 页（可融入 C 的末尾，无需单独成页）
[A] 方案   5-8 页（核心内容，按 MECE 分组）
[总结]     1 页
[Q&A]      1 页
```

**常见错误**：
- S 太长 — 听众已经知道的不需要花 5 页讲
- C 不够痛 — 如果冲突不够强烈，后面的方案就没有存在感
- A 缺乏结构 — 方案部分必须 MECE（互不重叠，完全穷尽）

## Sparkline 模型（Nancy Duarte） {#sparkline}

> 来源：《Resonate》

**适用场景**：理念传播型 — 传递信念、文化、愿景。

**核心机制**：在"现状"（What Is）与"愿景"（What Could Be）之间反复交替，制造张力和前进动力。

```
情感强度
  ^
  |    愿景     愿景      愿景
  |   /    \   /    \   /    \    --> 新常态
  |  /      \ /      \ /      \  /
  | /  现状   现状    现状      \/
  +---------------------------------> 时间
  开场                          行动号召
```

**结构模板**：

```
[开场]   描述现状（听众的日常现实）
[愿景1]  展示可能性（如果...会怎样？）
[现状1]  拉回现实（但现在我们面临...）
[愿景2]  更具体的未来画面（其他人已经...）
[现状2]  承认挑战（这需要我们...）
[愿景3]  完整的愿景（当我们做到时...）
[行动]   明确的 Call to Action（第一步是...）
```

**关键技巧**：
- 每次"现状→愿景"的跳跃都要比上一次更大
- 用具体故事/案例驱动，不是抽象概念
- 结尾的 Call to Action 必须具体可执行

**常见错误**：
- 全程只讲愿景 — 失去可信度
- 全程只讲现状 — 令人沮丧
- 交替不够明显 — 张力感消失

## ZUI 缩放模型 {#zui}

> Zoomable User Interface 思维，适用于复杂系统讲解。

**适用场景**：技术方案型 — 系统架构、技术全景、复杂流程。

**结构**：

```
[全景]    Zoom Out — 展示完整地图/架构/流程
[钻取1]   Zoom In  — 深入模块 A 的细节
[回溯]    Zoom Back — 回到全景，标记已讲区域
[钻取2]   Zoom In  — 深入模块 B 的细节
[回溯]    Zoom Back — 回到全景
...
[总结]    完整全景 + 各模块关系总结
```

**关键技巧**：
- 全景图在每次 Zoom Back 时复用，但高亮当前讲解区域
- Zoom In 时只聚焦一个模块，不要同时讲两个
- 用箭头/颜色表示数据流/调用关系

**页面预算**：全景 1 页 + 每个模块 2-3 页（含 Zoom Back）

## 金字塔结构（Barbara Minto） {#pyramid}

> 来源：《金字塔原理》

**适用场景**：成果汇报型 — 展示已完成的工作和成效。

**核心**：结论先行，分论点用 MECE 分组支撑。

```
            [核心结论]
           /     |     \
    [论据组1]  [论据组2]  [论据组3]
    /   \      /   \      /   \
  [数据] [数据] [数据] [数据] [数据] [数据]
```

**页面映射**：

```
[封面]     标题即核心结论
[总览]     3 个分论点概览（金字塔第二层）
[分论1]    2-3 页展开
[分论2]    2-3 页展开
[分论3]    2-3 页展开
[总结]     重申结论 + 下一步
```

**关键技巧**：
- 高管汇报永远结论先行 — 不要"先讲过程再给结论"
- 每组论据之间必须 MECE
- 总览页是最重要的一页，确保 3 个分论点能独立支撑结论

## 渐进揭示模型 {#progressive}

**适用场景**：培训教学型 — 教授新知识或技能。

**核心**：从已知到未知，每一步建立在前一步之上。

```
[锚点]     连接听众已知的概念
[概念1]    最基础的新概念
[练习1]    加固理解
[概念2]    在概念1基础上扩展
[练习2]    加固理解
...
[综合]     所有概念串联的完整画面
[实操]     听众可以立即动手的练习
```

**关键技巧**：
- 类比优先 — 用听众已知的概念解释未知
- 每个新概念后立即给出具体示例
- 复杂度应单调递增，不要来回跳

## 方法论速查表 {#cheatsheet}

| 模型 | 一句话本质 | 开场方式 | 结尾方式 |
|------|-----------|---------|---------|
| SCQA | 有问题，我有答案 | 共识事实 | 方案总结 + 行动 |
| Sparkline | 现实很痛，未来很美 | 描述现状 | 行动号召 |
| ZUI | 先看全貌，再看细节 | 全景地图 | 完整全景 |
| 金字塔 | 结论先行，证据支撑 | 核心结论 | 重申 + 下一步 |
| 渐进揭示 | 一步一步来 | 已知锚点 | 综合实操 |

### 组合使用

模型可以嵌套：
- **SCQA + ZUI**：用 SCQA 讲清问题，A 部分用 ZUI 展开技术方案
- **Sparkline + 金字塔**：用 Sparkline 驱动情感，每个愿景段内用金字塔组织论据
- **SCQA + 渐进揭示**：SQ 部分建立问题，A 部分用渐进揭示教方法
