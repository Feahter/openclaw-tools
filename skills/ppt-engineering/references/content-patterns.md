## 页面类型模板 {#page-types}

### 封面页

```
[大标题] 核心主张（非主题描述）
[副标题] 场景/时间/演讲者
[视觉]   极简，大面积留白

好: "微服务：先别急着拆"
坏: "微服务架构技术分享"
```

### 议程页（可选）

> 仅在 20 分钟以上演讲使用，5 项以内。

```
[标题] 今天聊什么
[列表] 3-5 个话题，当前高亮
```

### 核心观点页（最常用）

```
[标题] = 核心观点（一句话结论）
[视觉] 占 60%+ 面积（图表/示意图/代码）
[支撑] 最多 3 个要点（每个 3-7 词）
[备注] 演讲者口语笔记
```

### 数据冲击页

```
[大数字] 64px+ 字号，醒目颜色
[说明]   一句话解释这个数字意味着什么
[对比]   "是去年的 N 倍" / "相当于..."
```

### 对比页

```
[标题] 核心差异结论
[左栏] 方案 A / 之前
[右栏] 方案 B / 之后
[高亮] 关键差异用颜色标注
```

### 时间线页

```
[标题] 演进过程的核心结论
[时间线] 3-5 个关键节点
[高亮] 当前/关键节点用强调色
```

### 引用页

```
[引文] 24px+，带引号
[来源] 人名 + 身份
[视觉] 大面积留白，居中排版
```

### 总结页

```
[标题] 回扣 Throughline
[要点] 3 个核心 takeaway
[CTA]  明确的下一步行动
```

### Q&A 页

```
[标题] "Questions?"
[预设] 3 个可能被问到的问题 + 简要回答
[联系] 邮箱/微信（可选）
```

## 标题写作规范 {#titles}

> "标题应该是论点，不是标签。" — Barbara Minto

### 核心规则

**标题 = 这一页的核心结论**，读完所有页面标题应能串成完整故事。

| 类型 | 坏标题（描述性标签） | 好标题（核心观点） |
|------|---------------------|-------------------|
| 技术 | "系统架构" | "事件驱动解耦了部署依赖" |
| 数据 | "用户增长趋势" | "月活用户 6 个月增长 300%" |
| 方案 | "解决方案" | "缓存前置将 P99 延迟降至 50ms" |
| 对比 | "方案对比" | "方案 B 成本低 40% 且无迁移风险" |

### 标题自测

1. 遮住正文，只看标题 — 能否理解这页在说什么？
2. 串联所有标题 — 能否讲出完整故事？
3. 是否包含数字或具体信息？

## 数据叙事模式 {#data-narrative}

### 模式 1: 结论 → 证据

```
标题: "Q3 收入超目标 23%"（结论）
图表: 收入趋势图，目标线 + 实际线
注释: 直接标注超出部分
```

### 模式 2: 对比锚

```
标题: "处理速度提升至每秒 10 万条"
对比: "相当于 1 秒处理完整个《红楼梦》"
     "是竞品的 5 倍"
```

### 模式 3: 变化突出

```
标题: "部署频率从 1 次/周到 10 次/天"
视觉: Before/After 对比
高亮: 变化倍数用大字号 + 强调色
```

### 模式 4: 趋势+转折点

```
标题: "引入 CDN 后响应时间下降 80%"
图表: 时间线 + 标注转折点
注释: "这里我们做了 X"
```

## 粘性信息设计（SUCCESs） {#success}

> 来源：Chip & Dan Heath,《让创意更有黏性》

让核心信息被记住的 6 个要素：

### Simple（简单）

- 找到核心信息，删除其他一切
- "如果只能说一件事..." 测试
- 用类比降低理解门槛

**示例**：
- 坏："我们需要实现一个基于事件驱动的微服务架构以提升系统的可扩展性和部署独立性"
- 好："把一个大厨房拆成多个独立的小厨房，各炒各的菜"

### Unexpected（意外）

- 打破听众的预期模式
- 制造"知识缺口" — 让人意识到自己不知道什么
- 用"你以为...其实..."的结构

**示例**：
- "大多数人认为微服务能提高开发效率。但数据显示，60% 的团队在迁移后效率反而下降了。"

### Concrete（具体）

- 用看得见摸得着的描述替代抽象概念
- 数字 > 百分比 > 趋势词
- 案例 > 原则 > 理论

**示例**：
- 坏："显著提升了系统性能"
- 好："P99 延迟从 2.3 秒降到 47 毫秒"

### Credible（可信）

- 引用权威来源或数据
- 用内部真实数据而非行业平均
- "让听众自己验证"比"告诉他们答案"更有说服力

### Emotional（情感）

- 连接到听众个人利益："这意味着你不用再..."
- 用个人故事而非统计数据触发情感
- 关注"一个人"而非"一群人"

### Story（故事）

- 用时间线讲述变化过程
- 包含主角、挑战、解决、结果
- 开头设悬念，结尾给启发

## 过渡页设计 {#transitions}

### 页间逻辑连接词

| 关系 | 信号词 | 过渡方式 |
|------|--------|---------|
| 因果 | "因此..."、"这导致..." | 前页结论 = 后页前提 |
| 递进 | "不仅如此..."、"更重要的是..." | 层层加码 |
| 转折 | "但是..."、"然而..." | 制造张力 |
| 并列 | "同时..."、"另一方面..." | 分组呈现 |
| 总结 | "综合来看..."、"回到我们的问题..." | 收束回扣 |

### 分节过渡页

当演讲分为多个大段时，用简洁的过渡页标记段落切换：

```
[大标题] 段落名称
[副标题] 一句话预告接下来要讲什么
[视觉]   大面积留白 + 段落编号
```

## 常见反模式 {#anti-patterns}

### 内容反模式

| 反模式 | 问题 | 修正 |
|--------|------|------|
| 文字墙 | 每页 10+ 行文字 | 拆分为多页，每页 3 要点 |
| 描述性标题 | "项目背景" | 改为观点标题 "3 年技术债导致交付周期翻倍" |
| 无结论图表 | 放了图表不解释 | 标题写明图表的核心结论 |
| 全程念 PPT | 幻灯片=演讲稿 | 幻灯片=提示卡，备注区写讲稿 |
| 信息过载 | 一页塞 5 个概念 | 一页一信息，David JP Phillips 原则 |

### 视觉反模式

| 反模式 | 问题 | 修正 |
|--------|------|------|
| 彩虹配色 | 5+ 种颜色 | 回到 3 色规则 |
| 装饰性动画 | 旋转飞入弹跳 | 仅用 fade/slide |
| 字号过小 | <18px 正文 | 最小 24px（现场） |
| 裁切不当 | 图片被压缩变形 | 使用 object-fit: cover |
| Logo 堆砌 | 每页都有公司 Logo | 仅封面和尾页 |
