# skill-algo-researcher

算法深度研究专家。帮你快速理解任意算法的本质、实现细节和应用场景。

## 研究 SOP

### Step 1: 算法定性

先搞清楚这个算法"是什么"，用一句话定位它：

```
[算法名] = [本质类型] + [核心思想]

例：
Leiden = 图社区检测算法 + 三阶段保证连通性
AutoResearch = 自动化程序改进系统 + 贪心搜索代码空间
Frecency = 访问频率×时间衰减 + 优先级排序
```

**定性维度表：**

| 维度 | 要回答的问题 |
|------|-------------|
| 本质 | 属于哪类算法？（搜索/排序/图/优化/ML...） |
| 核心思想 | 一句话说清楚它在干什么 |
| 搜索类型 | 贪心/动态规划/回溯/随机/进化... |
| 时间复杂度 | O(?) |
| 空间复杂度 | O(?) |
| 解决什么问题 | 前置算法的什么缺陷？ |

### Step 2: 与经典算法对比

找到它的"家族"，理解它在哪里改进了什么：

```
┌─────────────────┬────────────────────┬─────────────────────┐
│ 经典算法        │ 相似点             │ 本算法改进点        │
├─────────────────┼────────────────────┼─────────────────────┤
│ [算法A]         │ [共同点]           │ [改进点]            │
│ [算法B]         │ [共同点]           │ [改进点]            │
└─────────────────┴────────────────────┴─────────────────────┘
```

**搜索方式：**
- 网络搜索：`[算法名] vs [相关算法]`、`[算法名] paper`
- 论文：arxiv、Google Scholar、Semantic Scholar
- 中文资源：知乎、CSDN、百度学术

### Step 3: 核心公式/数据结构

提取算法的数学本质：

**公式提取：**
```
核心公式 = 算法的"灵魂"
例：
Frecency: score = (1 + access_count) × time_multiplier(t)
Leiden ΔQ: k_i_in/(2m) - Σ_tot/(2m)² × (k_i + k_i_loops)
```

**关键数据结构：**
```
[数据结构名] — [用途] — [为什么选它]
例：
SparseMap — 社区权重存储 — 比普通 Map 省内存
SparseQueueSet — 节点遍历 — O(1) 入队出队 + 随机访问
```

### Step 4: 核心代码解析

找到算法的关键实现，逐段解读：

```bash
# 如果有 GitHub 项目，用 skill-project-butcher 先分析结构
# 然后定位核心文件

# 关键函数识别
grep -r "def \|function \|fn \|func " <core_file> | head -20

# 核心循环
grep -n "while\|for\|loop\|iterate" <core_file> | head -10
```

**代码解析模板：**
```
### [函数名]（[阶段名]）

[一句话说这个函数干什么]

```[语言]
[关键代码片段]
```

**关键**：[最重要的一行或一个设计决策，解释为什么这样做]
```

### Step 5: 优化方向分析

| 当前实现 | 可优化方案 | 适用场景 |
|---------|-----------|---------|
| [现有策略] | [改进方案] | [什么时候用] |

**常见优化维度：**
- 时间复杂度：O(n²) → O(n log n)
- 空间复杂度：减少内存占用
- 并行化：串行 → 多线程/分布式
- 精度/速度权衡：精确 → 近似算法
- 适应性：固定参数 → 自适应参数

### Step 7: 实现代码

根据理解，输出两个版本：

**基础版**（清晰易懂，适合学习）：
```[语言]

## 注意事项

1. **先定性再深入** — 不要一上来就看代码，先搞清楚算法的本质
2. **找对比对象** — 理解一个算法最好的方式是找它改进了什么
3. **公式是灵魂** — 核心公式往往就是算法的全部
4. **两版实现** — 基础版学习，优化版使用
5. **记录到 memory** — 研究结果存到 `memory/research/algorithms/<算法名>/README.md`
