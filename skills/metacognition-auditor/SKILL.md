# Metacognition Auditor — 元认知审计器

*对认知模型的认知 — 二阶思维执行层*

---

## 二阶思维 vs 一阶思维

| 维度 | 一阶思维 | 二阶思维 |
|------|---------|---------|
| 核心问句 | "系统的反馈回路是什么？" | "我预设的边界条件是什么？为什么选这个粒度？" |
| 关键动作 | 映射系统结构 | 检视认知盲区（Unknown Unknowns） |
| 风险点 | 局部最优 | 分析瘫痪（overthinking） |

---

## 适用场景

### 场景1：技术选型
```
一阶：这个框架性能好（benchmark领先）
二阶：这个框架3年后还是最优选择吗？团队认知负荷如何？
```

### 场景2：故障复盘
```
一阶：修复了这个Bug
二阶：为什么当初会设计这个缺陷？是决策机制的问题吗？
```

### 场景3：商业决策
```
一阶：计算CAC/LTV
二阶：对手的反应函数是什么？监管套利是否可持续？
```

## 预验尸（Pre-mortem）协议

**执行时机**：任何重大决策实施前

**步骤**：
1. 假设项目/决策已经失败
2. 倒推：是什么导致了失败？
3. 识别：哪个认知盲区让我们做出了这个决定？
4. 预防：现在能做什么来避免？

**问句**：
- "项目失败了，最可能的原因是什么？"
- "我们忽略了什么警告信号？"
- "现在做的什么假设3年后会被证明是错的？"

---

## 认知盲区识别框架

### 四种 Unknown Unknowns

| 类型 | 描述 | 检测方法 |
|------|------|---------|
| **已知的未知** | 知道有风险，但未量化 | 风险清单 |
| **未知的已知** | 知道，但没意识到自己知道 | 专家访谈 |
| **未知的未知** | 不知道存在 | 预验尸、情景规划 |
| **元级盲区** | 认知框架本身的局限 | 二阶审视 |

### 认知偏见清单

| 偏见 | 表现 | 缓解方法 |
|------|------|---------|
| 确认偏误 | 只看支持当前观点的信息 | 强制为对立观点辩护 |
| 过度自信 | 低估风险，高估成功率 | 预验尸 + 历史数据对比 |
| 沉没成本 | 不愿放弃失败决策 | "如果没有这项投入，还会继续吗？" |
| 近期偏差 | 过度重视最新信息 | 长周期数据加权 |
| 群体思维 | 团队内不愿反对主流意见 | 匿名意见收集 |

## 决策审计框架

### 三问审计

对每个重要决策，强制回答：

1. **我为什么做这个决定？**
   - 证据是什么？
   - 反例是什么？
   - 3年后回头看，这个决定还成立吗？

2. **如果我错了，可能在哪里？**
   - 最大风险是什么？
   - 什么信号会告诉我错了？
   - 我愿意承受多大损失？

3. **谁会因为这个决定受苦？**
   - 直接影响
   - 间接影响
   - 长期影响

## 使用工具

### 1. 预验尸执行

```python
from metacognition_auditor import pre_mortem

result = pre_mortem(
    decision="采用微服务架构",
    timeline="6个月后",
    participants=["架构师", "运维", "开发"]
)
# 输出：失败原因假设、认知盲区、预防措施
```

### 2. 认知偏见检测

```python
from metacognition_auditor import detect_bias

result = detect_bias(
    decision="继续投入当前技术栈",
    evidence=["团队熟悉", "已有投资", "性能足够"],
    opposing_view=["3年后技术债累积", "招人困难"]
)
# 输出：偏见类型、风险评分、建议
```

### 3. 决策审计

```python
from metacognition_auditor import audit_decision

result = audit_decision(
    decision="选择方案A",
    alternatives=["方案B", "方案C"],
    criteria=["成本", "风险", "收益"]
)
# 输出：决策质量评估、潜在盲区、改进建议
```

## 输出格式

### 预验尸报告

```markdown
## 预验尸分析：[决策名称]

**假设时间点**：X个月后
**假设结果**：❌ 失败

### 失败原因（按可能性排序）
1. [最可能原因]
2. [次可能原因]

### 认知盲区
- 我们以为：[假设]
- 实际上：[盲区]

### 预防措施
1. 现在就做：[行动]
2. 建立信号：[监控指标]
```

### 决策审计报告

```markdown
## 决策审计：[决策名称]

**审计时间**：[时间]
**决策者**：[角色]

### 一阶分析
- 目标：[是什么]
- 证据：[支持决策的证据]

### 二阶审视
- 边界条件：[这个决策的前提]
- 风险假设：[可能错的假设]
- 反例：[支持对立面的证据]

### 认知盲区
- 检测到：[偏见类型]
- 影响评估：[影响程度]

### 建议
1. [如果接受此决策]
2. [如果拒绝此决策]
```

## 局限与警示

- **分析瘫痪风险**：二阶思维成本高，仅用于高 stakes 决策
- **主观性**：盲区识别依赖自我审视，可能遗漏
- **过度哲学化**：避免在执行层滥用元认知

**使用原则**：重大决策 + 高不确定性 + 时间允许 → 启用元认知审计
