*完整研究流程。SKILL.md RESEARCH 模式触发后，按本文件执行。*

---

## 核心理念

算法研究 = 理解原理 + 提炼模式 + 归档复用。不是读完论文就结束，而是把算法变成可复用的思维单元。

## 七阶段流程

| 阶段 | 名称 | 产出 | 质量门槛 |
|------|------|------|---------|
| S1 | 发现 | 起源标注 | 算法有明确名字和类别 |
| S2 | 定位 | 算法定性表 + 竞品对比 | 一句话说清解决什么问题 |
| S3 | 原理理解 | 核心公式 + 伪代码 | 能向不懂的人解释为什么有效 |
| S4 | 源码分析 | 关键路径拆解 + 数据结构选型分析 | 能说出为什么用这个结构 |
| S5 | 泛化提取 | 可迁移模式卡 + 适用条件卡 | 2+ 个不同领域的迁移场景 |
| S6 | 场景映射 | 当前项目匹配 + 风险评估 | 有具体问题匹配，非泛泛而谈 |
| S7 | 归档入库 | algorithms.md 条目 + 研究文档 | 模板完整、代码可运行、评价客观 |

## S3 原理理解 — 四维拆解

### 1. 核心公式
用数学语言描述算法的关键计算。无法公式化的改为描述决策过程。

### 2. 关键机制拆解
```
| 特征 | 实现方式 | 为什么这样设计 |
|------|---------|--------------|
|      |         |               |
```

### 3. 正确性保证
算法在什么条件下保证正确？有没有反例？

### 4. 局限性
- 时间/空间复杂度上界
- 什么情况下会退化
- 典型失败案例

## S4 源码分析 — 关键路径拆解

**分析顺序**：1) 数据结构选型 → 2) 主循环/递归逻辑 → 3) 边界条件 → 4) 优化路径

**代码注释规范**：
```python
# 关键行注释格式：
# [行号] 目的：这样设计的原因
# 例如：
# 12  压缩：避免重复构建相同子结构
# 34  缓存：用空间换时间，记录已计算结果
```

## 错误分级标准

| 级别 | 定义 | 处理 |
|------|------|------|
| P0 致命 | 复杂度分析错误、正确性保证说错 | 必须修正，附反例 |
| P1 严重 | 适用条件遗漏、源码关键路径误读 | 补充完整，注明局限 |
| P2 一般 | 泛化场景不足 | 补充更多场景 |
| P3 建议 | 格式不规范 | 改进模板合规性 |
