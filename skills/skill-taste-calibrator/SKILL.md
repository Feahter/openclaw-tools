# skill-taste-calibrator

品味与判断力持续校准工具。

## 数据目录

```
~/.openclaw/workspace/memory/research/taste-judgement/
├── README.md              — 总览和统计
├── log/judgement-log.md   — 判断日志
├── principles/principles.md — 提炼原则
└── calibration/           — 月度校准
```

## 四种操作

### 1. 记录判断（最常用）

用户说"我觉得XXX"、"帮我记录"时触发。

**追加到 `log/judgement-log.md`：**
```markdown
### [YYYY-MM-DD] | [审美/商业/人] | [对象名]
- **判断**：[明确的判断，不能模糊]
- **理由**：[为什么这么判断]
- **待验证**：[什么时候可以验证，验证什么]
- **置信度**：[高/中/低]
```

**原则：判断必须明确可验证**
- ❌ "感觉还不错"
- ✅ "会在 2 年内失败，因为..."

---

### 2. 校准验证

用户说"验证一下"、"回看判断"时触发。

**步骤：**
1. 读取 `log/judgement-log.md`，找到待验证的条目
2. 搜索/询问当前状态
3. 记录验证结果：
```markdown
- **验证时间**：[日期]
- **实际结果**：[发生了什么]
- **判断是否准确**：✅ 准确 / ❌ 错误 / ⚠️ 部分准确
- **错在哪里**：[如果错了，分析原因]
```
4. 更新 `README.md` 中的统计数字

---

### 3. 提炼原则

月度校准时，或积累了 5+ 条验证记录后触发。

**步骤：**
1. 读取所有已验证的判断
2. 找规律：哪类判断总是对的？哪类总是错的？
3. 提炼成原则，追加到 `principles/principles.md`：
```markdown
### 原则 [编号]：[原则标题]
- **内容**：[具体原则]
- **来源**：[哪次判断得出的]
- **适用场景**：[审美/商业/人]
- **反例**：[什么情况下不适用]
```

---

### 4. 月度校准报告

每月末触发，生成 `calibration/YYYY-MM.md`：

```markdown
# [月份] 校准报告

## 本月判断统计
- 新增判断：N 条
- 已验证：N 条
- 准确率：N%

## 准确的判断
[列出]

## 错误的判断
[列出 + 分析原因]

## 本月提炼原则
[新增原则]

## 下月重点
[需要重点训练的判断类型]
```

## 高质量样本输入建议

### 审美
- 每周系统看 1 个设计师/艺术家完整作品集
- 推荐：Dieter Rams、Paul Rand、Jony Ive、原研哉
- 看完写 100 字：好在哪里，用了什么手法

### 商业
- 追踪 5-10 个公司/产品，定期写判断
- 读：《Poor Charlie's Almanack》、Stratechery、Howard Marks 备忘录
- 关注：产品发布、融资、倒闭事件，每次都写判断

### 人
- 接触新人后立即记录第一印象
- 3-6 个月后回看
- 关注：言行一致性、压力下的反应、对待弱者的方式

## 注意事项

1. **判断要明确** — 模糊的判断无法校准
2. **理由要具体** — "感觉好"不算理由
3. **置信度要诚实** — 不确定就写低
4. **错了不可怕** — 错了才有校准价值
5. **原则要有来源** — 没有来源的原则是空话
