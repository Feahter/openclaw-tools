# skill-repo-research

薄编排层：判断用户需求，分发给项目屠夫（代码分析）和项目猎手（技术情报）。

## 协作流程

```
用户输入（GitHub URL 或本地路径 + 研究意图）
  │
  ▼
Step 1: 意图判断
  │
  ├─ "代码怎么组织的"、"怎么用"、"帮我改" → 只用屠夫，跳过猎手
  ├─ "这项技术是什么"、"值不值得学"、"生态" → 只用猎手，跳过屠夫
  └─ "全面了解"、"深度研究"、"学习这个项目" → 两者都用，先猎手后屠夫
  │
  ▼
Step 2: 执行子 skill（见下方指令）
  │
  ▼
Step 3: 合并输出
  - 猎手输出 → 技术定位 + 生态 + 可复用代码析出
  - 屠夫输出 → 代码架构 + 技术栈 + 快速上手
```

## 子 skill 调用指令

### 屠夫（代码分析）
激活后直接使用 skill-project-butcher 的完整流程。

### 猎手（技术情报）
激活后使用 skill-tench-hunter 的完整流程。
