# Bazi-Graph：八字知识图谱 + Context Builder

> ⚠️ 注意：本 skill 混合了八字（四柱）和紫微斗数两套体系。
> 核心规则以八字为主；命宫/身宫/官禄宫等属于紫微斗数概念。

## 已完成模块

### conflict_detector.py ✅
独立运行的冲突检测模块，检测四大矛盾信号：

```bash
python3 skills/bazi-graph/conflict_detector.py <年> <月> <日> <时>
```

**检测信号**：
| 信号 | 严重度 | 说明 |
|------|--------|------|
| 官杀混杂 | 🔴 高 | 正官+七杀同现，女命感情乱，男命小人多 |
| 财星坏印 | 🟠 中高 | 财克印，贪财坏印主：为利弃义、学历中断 |
| 身旺无财/弱 | 🟡 中 | 日主得令但无财，有才缺资源 |
| 枭神夺食 | 🟠 中高 | 福气被夺，主抑郁、食禄受损 |

**无矛盾时**：✅ 无明显矛盾信号，格局较为纯净

**使用示例**：
```bash
python3 skills/bazi-graph/conflict_detector.py 1990 3 15 14
python3 skills/bazi-graph/conflict_detector.py 1988 5 20 10
```

## 节点类型（V）

### V1：天干
```
{甲, 乙, 丙, 丁, 戊, 己, 庚, 辛, 壬, 癸}
属性：
  - 五行：木/火/土/金/水
  - 阴阳：阳/阴
  - 方位：东/南/中/西/北
```

### V2：地支
```
{子, 丑, 寅, 卯, 辰, 巳, 午, 未, 申, 酉, 戌, 亥}
属性：
  - 五行：见地支五行表
  - 阴阳：见地支阴阳表
  - 地道：藏干（三杰）
```

### V3：十神
```
{比, 劫, 食, 伤, 财, 才, 官, 杀, 枭, 印}
关系定义：
  - 以日干为基准，按五行生克推导
  - 例：日干甲，见辛官，见庚杀，见乙劫
```

### V4：宫位
```
{命宫, 身宫, 夫妻宫, 子女宫, 财帛宫, 疾厄宫, 迁移宫, 官禄宫, 田宅宫, 福德宫, 父母宫}
属性：
  - 所在地支
  - 主星
  - 辅星
```

### V5：运限
```
{大运, 流年, 流月, 流日, 流时}
属性：
  - 起运年龄
  - 所属五行
  - 强弱
```

### V6：神煞
```
{贵人, 驿马, 文昌, 天乙, 桃花, 魁罡, ...}
属性：
  - 喜忌
  - 触发条件
```

## 边类型（E）

### E1：五行关系（必选项）
```
生：木生火 → 火生土 → 土生金 → 金生水 → 水生木
克：木克土 → 土克水 → 水克火 → 火克金 → 金克木
冲：子午冲, 丑未冲, 寅申冲, 卯酉冲, 辰戌冲, 巳亥冲
合：子丑合, 寅亥合, 卯戌合, 辰酉合, 巳申合, 午未合
刑：三刑（寅巳申, 丑戌未, 子卯, 辰辰, 午午, 申申...）
害：六害（子未, 丑午, 寅巳, 卯辰, 申亥, 酉戌）
```

### E2：十神推导规则
```
日干 + 五行 → 十神
例如：日干甲木
  - 遇辛金 → 正官（七杀为庚）
  - 遇癸水 → 印（枭为壬）
  - 遇乙木 → 比肩（劫财为甲）
```

### E3：透藏规则
```
透干：四柱中天干出现某五行
藏干：地支中地支藏干包含某五行
会：月令地支与其它地支发生三合局/三会局
```

## Context Builder 场景

### 场景1：explain（推理解释）
```
输入：一个八字 + 一个结论
输出：推理路径（哪些节点+边导致这个结论）

例：
输入：「此命事业强，为什么？」
输出：
  1. 官禄宫（巳）→ 丙火 → 食神星
  2. 食神 → 创造力 + 事业心
  3. 大运癸丑 → 丑中藏癸（印）+ 辛（官）
  4. 印星护官 → 事业得到官方认可
  结论路径：食神旺 → 事业心强 → 官运佳
```

### 场景2：verify（冲突检测）
```
输入：一个八字
输出：局中矛盾信号

例：
  - 局中同时出现「官杀混杂」（正官+七杀同透）
  - 日主弱但财星旺
  - 印星被财星坏

这些冲突信号 → 提醒用户重点关注
```

### 场景3：onboard（新命例快速理解）
```
输入：新八字
输出：
  - 整体格局（从旺/假从/普通）
  - 最强的三个能力维度
  - 最弱的三个风险维度
  - 建议重点研究的推理路径
```

### 场景4：diff（对比分析）
```
输入：两个八字 or 两个时辰
输出：关键差异点

例：
  - 戊时 vs 己时：前者偏阳，后者偏阴
  - 差异点在「事业宫」vs「夫妻宫」
```

## 图谱存储格式

使用 JSON/JSONL 格式，存储在：
```
memory/bazi/graph/
  schema.json        # 节点/边类型定义
  entities/          # 具体实体
    tiangan.json     # 十个天干
    dizhi.json       # 十二地支
    shishen.json     # 十神关系表
  rules/             # 推理规则
    wuxing.json      # 五行生克冲合
    mingpan.json     # 命盘规则
  cases/             # 命例图谱
    {case_id}.json   # 每个命例的图谱
```

## 与现有八字引擎的关系

现有八字引擎（`skills/bazi/`）：
- 强于：排盘、十神计算、大运推演
- 弱于：推理过程不透明、无法追问「为什么」

Bazi-Graph 的价值：
- 给现有引擎加上「推理可解释层」
- 不替代现有引擎，是增强层
- 用户可以同时使用两种服务
