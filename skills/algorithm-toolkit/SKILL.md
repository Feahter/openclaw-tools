## 算法分析框架

当用户要求"分析"、"剖析"或"理解"一个算法时，使用以下框架：

### 1. 算法定性

```
| 维度 | 分析 |
|------|------|

| 核心思想 | 主要机制 |
| 时间复杂度 | O() |
| 空间复杂度 | O() |
```

### 2. 与经典算法对比

```
┌─────────────────┬────────────────────┬─────────────────┐
│ 经典算法        │ 相似点             │ 改进点          │
├─────────────────┼────────────────────┼─────────────────┤
│ LRU             │ 最近访问优先       │ 连续访问加分    │
│ 指数衰减        │ 时间越近权重越高   │ 分段系数        │
│ PageRank        │ 迭代更新           │ 每次访问+1      │
└─────────────────┴────────────────────┴─────────────────┘
```

### 3. 核心公式

用数学公式描述算法的核心计算

### 4. 关键算法特征

```
| 特征 | 实现 |
|------|------|
| 访问追踪 | 单调递增计数器 |
| 时间衰减 | 离散分段 |
| 匹配算法 | 子串顺序匹配 |

```

### 5. 优化方向

```
可优化点:
- 用堆维护 Top-K → O(log n)
- 分数用对数压缩 → 避免大数问题
- 按时间分桶 → 快速筛选
```

### 6. 伪代码模板

```python
def calculate_score(entry):
    # 1. 计算基础分
    base = entry.access_count
    
    # 2. 计算时间衰减
    elapsed = now - entry.last_access
    multiplier = get_multiplier(elapsed)
    
    # 3. 返回最终分数
    return base * multiplier

def get_multiplier(elapsed):
    if elapsed < 1h: return 4
    elif elapsed < 1d: return 2
    elif elapsed < 1w: return 0.5
    else: return 0.25
```

## 数据结构速查

### 并查集 (Union-Find)
- **用途**：连通性判断、最小生成树
- **核心**：路径压缩 + 按秩合并
- **复杂度**：O(α(n)) 近似 O(1)

### 线段树 (Segment Tree)
- **用途**：区间查询、区间修改
- **核心**：分治思想，树形结构
- **复杂度**：O(log n)

### 树状数组 (Fenwick Tree/BIT)
- **用途**：前缀和、单点修改
- **核心**：lowbit 技巧
- **复杂度**：O(log n)

### LRU 缓存
- **用途**：最近最少使用淘汰
- **核心**：哈希 + 双向链表
- **复杂度**：O(1)
