# 小说创作SOP - 严格版 v3.0
## 基于「种子→萌芽→枝干→枝叶」模型

## 核心理念

小说创作如同种植一棵故事之树：
```
种子（概念）→ 萌芽（大纲）→ 枝干（分镜）→ 枝叶（内容）
```

| 层级 | 产出 | 核心任务 |
|------|------|---------|
| **种子** | 核心概念卡 | 确定书名、类型、金手指、爽点、主题 |
| **萌芽** | 主线剧情树 | 规划主线、冲突、人物弧光、转折点 |
| **枝干** | 卷纲+章纲 | 设计章节结构、场景安排 |
| **枝叶** | 正文+摘要 | 填充具体内容、章节写作 |

---

## 触发条件

当用户要求以下操作时使用此Skill：
- 继续小说创作
- 创作新章节
- 从零开始规划新小说
- 修复创作流程问题
- 批量生成章节

---

## 📋 核心原则

| 原则 | 说明 |
|------|------|
| **每步必保存** | 完成任何步骤立即保存进度 |
| **错误不过夜** | 出错立即记录，继续下一章 |
| **层级清晰** | 每个创作阶段有明确产出物 |
| **锁文件超时** | 超过30分钟自动清理 |
| **幂等执行** | 同一章节可重复执行 |

## 🔄 四层创作流程

### L1 种子层：概念确定（项目启动时执行一次）

**触发**：创建新小说时

**产出**：核心概念卡

```

书名：《》
类型：都市重生/系统流/克苏鲁/...
核心金手指：
  - 系统名称：
  - 系统功能：
  - 能力限制：
核心爽点：
  - 爽点1：
  - 爽点2：
  - 爽点3：
目标读者：
核心主题（一句话）：
更新时间：
```

**执行步骤**：
1. 与用户确认核心要素
2. 生成核心概念卡
3. 保存到 {小说目录}/核心概念卡.md

**触发**：开始新一卷创作时

**产出**：大纲 + 人物档案 + 卷纲

```

  - 主冲突：谁阻止主角？为什么？
  - 次冲突：有哪些障碍？

  - 主角：从xxx到xxx

  - 反派：从xxx到xxx

  1. 第N章：xxx事件 → 格局升级
  2. 第N章：xxx事件 → 冲突加剧
  3. ...

伏笔线索：
```

### L3 枝干层：分镜设计（每章创作前执行）

**触发**：创作每一章之前

**产出**：章节规划 + 分镜脚本

> **📌 分级执行机制（v4.0 新增）**
> 
> | 章节类型 | 分镜字数 | 规划字数 | 适用场景 |
> |---------|---------|---------|---------|
> | **日常章** | 200字 | 150字 | 铺垫、过渡、信息交代 |
> | **普通章** | 350字 | 250字 | 大部分剧情章节 |
> | **关键章** | 500字 | 300字 | 转折点、高潮、重大事件 |
> 
> **判断规则**：
> - 章节涉及「重大冲突」「角色死亡」「情感转折」「爽点爆发」→ 关键章
> - 章节是「首次登场」「重要配角」「新场景」→ 普通章
> - 其他 → 日常章

#### 3.1 章节规划（分级字数）
```
【chXXX】章节规划
【类型】：日常章/普通章/关键章

【起】(约700字)
- 场景：
- 人物：
- 目标：

【承】(约1000字)
- 阻碍：
- 行动：

【转】(约1000字)
- 意外：
- 升级：

【合】(约500字)
- 解决：
- 结尾悬念：
```

#### 3.2 分镜脚本（含衔接模板 + 爽点强制嵌入）
```
【chXXX】分镜脚本

上一章结尾: {精确描述最后一幕}
↓ 发生了什么:{简述时间跳跃/过渡}
本章开头: {场景+人物+行动}

【过渡模板】- 必选其一
□ 模板A-时间过渡: "X天后" + 感受/铺垫
□ 模板B-场景过渡: 情绪延续 + 空间转移 + 新事件
□ 模板C-因果过渡: 上一章结果 + 连锁反应 + 本章行动

【矛盾设计】- 必做
- 本章核心冲突: {是什么}
- 主角当前困境: {处境多糟？不行动的后果？}
- 推动力: {必须行动的理由}
- 后果: {行动失败/不行动会怎样？}

【爽点强制嵌入】- 必做（关键章必填，日常章至少1项）
- 爽点类型: 打脸/逆袭/揭露/反杀/情感突破/技术展示/其他
- 爽点位置: 章开头/中段/高潮/结尾
- 爽点呈现: {具体描写方式}
- 爽点效果: {读者预期情绪反应}

【创意与记忆点检查】- 必做（v4.1 新增）
> 灵感来源：《宇宙super英雄传》ch001 成功分析 - 创意炸点+情绪锚点是第一章成功的关键

| 检查项 | 检查内容 | 不合格表现 | 合格示例 |
|--------|---------|-----------|---------|
| **开场Hook** | 开头有没有抓人点？ | "时间-地点-人物"流水账，太平淡 | 对话引入、悬念、意外、动作 |
| **创意亮点** | 这一章的核心创意是什么？ | 说不出来，纯推进剧情 | 克苏鲁→弹幕、烤红薯反差 |
| **记忆点** | 有没有让读者记住的细节？ | 看完就忘 | "太空酒吧吃烤红薯" |
| **情绪曲线** | 有没有两种以上情绪交替？ | 全程一个调 | 逗比→神秘→搞笑→悬疑 |
| **人物出场** | 新人物怎么出场？ | "XX是XX帝国的公主"念设定 | 通过行动/对话/冲突展现 |

**自检问题**：
1. 这一章的核心画面是什么？能用一句话描述出来吗？
2. 没有文字，读者能"看到"什么？
3. 对手/反派的反应是什么？
4. 普通人会怎么想这个场景？

**质量门槛**：任何一项不合格 → 退回重写

关键衔接点:
- 角色在哪: {位置连续性}
- 剧情因果: {为什么发生}
- 主线绑定: {与核心冲突的关联}

本章结局悬念: {为下一章留钩子}
```

### 衔接模板详解

#### 模板A：时间过渡
适用场景：时间跳跃（第二天、三天后、几小时后）

```
【承接】上一章结尾 + 过渡句 + 本章开头

公式：
"【时间词】+【感受/状态】" → "【新事件/新场景】"

示例：
"战斗结束的第三天，陈默躺在床上浑身酸痛。"
"三天足够让伤口结痂，却不够让他忘记那晚的眼神。"
"手机响起，是林清雅的专属铃声。"

要点：
- 时间词要具体（"三小时"比"不久"好）
- 过渡句要有关联（感受承接上一章情绪）
- 本章开头立即有新事件
```

#### 模板B：场景过渡
适用场景：空间切换（会议室→电梯、战场→家中）

```
【承接】情绪延续 + 空间转移 + 新事件

公式：
"【情绪动作】" → "【移动过程】" → "【新事件】"

示例：
"陈默握紧拳头，指甲陷入掌心。"
"走出会议室，他直接进了电梯。"
"电梯门关上的瞬间，手机震动了。"

要点：
- 情绪不能断（带着上一章情绪离开）
- 空间转移要简洁（一句话带过）
- 新事件要立即抓住注意力
```

#### 模板C：因果过渡
适用场景：事件连续（做完一件事→做下一件）

```
【承接】上一章结果 + 连锁反应 + 本章行动

公式：
"【决定/结果】" → "【连锁反应】" → "【新行动】"

示例：
"既然已经暴露，就没必要再装下去。"
"陈默打开系统面板，开始最后的布局。"
"A级任务触发：72小时内获取微软核心代码。"

要点：
- 因果要清晰（上章→本章的逻辑）
- 连锁反应自然（不是突然冒出来的）
- 本章行动立即开始
```

## 🔄 完整执行流程（17步）

### Step 1: 检查锁文件
```
检查 {小说目录}/.lock 是否存在
├── 存在 → 检查修改时间
│   ├── 超过30分钟 → 删除并继续
│   └── 未超过30分钟 → 报错退出
└── 不存在 → 继续
```

### Step 2: 读取进度
```
读取 {小说目录}/progress.json
├── 解析 current_chapter
├── 确定当前层级（种子/萌芽/枝干/枝叶）
└── 计算下一任务
```

### Step 3: 检查层级状态
```
根据当前进度，判断需要执行哪个层级：
├── L1 种子层：核心概念卡不存在 → 执行L1
├── L2 萌芽层：需要新卷大纲 → 执行L2
├── L3 枝干层：需要章节分镜 → 执行L3
└── L4 枝叶层：需要正文填充 → 执行L4
```

### Step 4: 创建锁文件
```
创建 {小说目录}/.lock
内容: 创作中\n层级:{L?}\nPID:{PID}\n时间:{时间}\n章节:ch{编号}
```

### Step 5: 加载上下文
```
必读:
1. 核心概念卡（如有）
2. 大纲.md
3. 人物档案.md
4. ch{上一章}_summary.md
```

### Step 6-9: 执行对应层级

**L1 种子层**：
- 与用户确认核心要素
- 生成核心概念卡
- 保存

**L2 萌芽层**：
- 生成立线剧情树
- 生成/更新人物档案
- 生成本卷卷纲

**L3 枝干层**：
- 生成分镜脚本（必做）
- 生成章节规划
- 矛盾设计检查（必做）

**L4 枝叶层**：
- 生成正文
- 生成章节摘要
- QA审稿（含衔接检查）

### Step 9: QA审稿（含衔接检查）- 必做

**检查项目：**
- 衔接度 (与上一章)
- 创新度 (避免套路重复)
- 完整度 (起承转合)
- 张力检测 (冲突是否足够？主角处境够糟吗？)
- **衔接检查 (以下5项必检)**

#### QA衔接检查清单（5项必检）

| # | 检查项 | 问题表现 |
|---|--------|---------|
| 1 | **场景连续性** | 突然换地图无过渡 |
| 2 | **时间连续性** | 时间跳跃无过渡句 |
| 3 | **情绪连续性** | 情绪突然180度转弯 |
| 4 | **因果连续性** | 事件凭空出现 |
| 5 | **信息连续性** | 新人物/物品无介绍 |

#### 衔接问题严重度分级

| 等级 | 问题 | 处理 |
|------|------|------|
| 🔴 严重 | 场景/时间突然跳跃无过渡 | **必须修改** |
| 🟠 中等 | 情绪断崖式变化 | 建议修改 |
| 🟡 轻微 | 信息介绍稍显突兀 | 可以不改 |
| 🟢 通过 | 衔接自然流畅 | 通过 |

#### 衔接问题修复模板

**问题：场景跳跃**
```
修复：加入场景转移句
原文：打完架后，他回到家。
修复：打完架后浑身是血的陈默，踉跄着走出巷子。半小时后，他终于回到了家。
```

**问题：时间断裂**
```
修复：加入时间过渡句
原文：第二天，陈默去找林清雅。
修复：战斗结束后的第二天，伤口还在隐隐作痛。陈默拨通了林清雅的电话。
```

**问题：情绪断层**
```
修复：加入情绪缓冲
原文：听到这个消息，陈默很开心。他然后去吃饭了。
修复：听到这个消息，陈默愣了一下，随即笑了。笑了三秒后，他深吸一口气，压下情绪。现在不是高兴的时候，还有正事要办。
```

### Step 10: 更新进度
```
更新 progress.json:
- current_level: L?
- current_chapter: chXXX
- chapter_status: completed/in_progress
```

### Step 11: 保存日志
```
追加到 创作日志.md
```

### Step 12: 清理锁文件
```
删除 {小说目录}/.lock
```

## ⚠️ 错误处理清单

| 错误类型 | 处理方式 | 是否阻塞 |
|----------|---------|---------|

| 进度保存失败 | 强制保存 | ❌ 不阻塞 |
| 网络中断 | 保存状态，重试 | ❌ 不阻塞 |

## 🚀 使用示例

### 示例1: 继续创作（现有小说）

```
用户: 继续创作重生带AI夯爆了

执行:
1. 检查锁文件
2. 读取progress.json → 当前ch101，L4枝叶层
3. 检查ch102是否存在
```

### 1. 卷末钩子设计（来自《宿命之环》）

> 每卷结尾的"卷末效应"成为读者追更的核心动力

#### 卷末钩子类型

| 类型 | 描述 | 示例 |
|------|------|------|
| **悬念型** | 结尾揭示关键信息但不说 | "他终于知道了自己的真实身份，但是..." |
| **危机型** | 主角陷入绝境或被发现 | "刀已经架在脖子上了" |

| **抉择型** | 主角面临重大选择 | "救她，还是救全城人？" |

#### 卷末钩子模板

```
【卷末钩子】- 每卷最后3章必做

卷末第3章：埋设伏笔 + 情绪铺垫
卷末第2章：冲突升级 + 选择出现
```

### 1. 百世书/读档流系统（来自《苟在初圣魔门当人材》）

> 死后重开一世，可带回前世宝物、修为、寿命或觉醒天赋

#### 读档流系统设计

```
【百世书系统】- 机制创新必做

核心机制：
1. 【死亡触发】死亡后自动重开一世
2. 【带回资源】可选带回：
   - 宝物（随机1-3件）
   - 修为（部分保留）
   - 寿命（部分保留）
   - 天赋（觉醒新天赋）
3. 【一世限制】每世有寿命/境界上限
4. 【累计优势】多世积累形成碾压

爽感闭环：
死亡 → 重开 → 变强 → 再次死亡 → 变得更强
= "死亡即成长"的爽感

设计要点：
- 死亡要有意义（不能白死）
- 每世要有成长感
- 多世后要有"终极形态"
```
