# 小说创作SOP - 严格版 v4.2
## 基于「种子→萌芽→枝干→枝叶」模型 + Kimi故事结构方法论

## 核心理念

**双模型驱动**：
1. **种子模型**：创作流程管理（种→萌芽→枝干→枝叶）
2. **故事结构**：内容质量保证（五要素→三幕→节奏）

```
种子（概念）→ 萌芽（大纲）→ 枝干（分镜）→ 枝叶（内容）
   ↓              ↓            ↓           ↓
故事结构 → 五要素 → 三幕结构 → 节奏测试
```

| 层级 | 产出 | 核心任务 |
|------|------|---------|
| **种子** | 核心概念卡 | 确定书名、类型、金手指、爽点、主题 |
| **萌芽** | 主线剧情树 | 规划主线、冲突、人物弧光、转折点 |
| **枝干** | 卷纲+章纲 | 设计章节结构、场景安排 |
| **枝叶** | 正文+摘要 | 填充具体内容、章节写作 |

## 📋 创作流程（17步）

### 阶段1：种子（1-3步）
1. **概念确立** → 确定书名、类型（金手指、爽点类型）
2. **主题锚定** → 核心冲突、终极问题
3. **人物原型** → 主角/反派/导师的人设卡

### 阶段2：萌芽（4-7步）
4. **主线剧情树** → 主线+支线+伏笔
5. **五要素闭环** → 人物目标+冲突类型+结构设计+视角选择+主题
6. **三幕结构** → 设定(25%)→对抗(50%)→解决(25%)
7. **转折点设计** → 触发/中点/崩溃/灵魂黑夜

### 阶段3：枝干（8-12步）
8. **卷纲设计** → 多少卷、每卷主题
9. **章纲设计** → 每章核心事件
10. **场景卡** → 关键场景详细设计
11. **衔接设计** → 章之间怎么接（A/B/C模板）
12. **爽点埋入** → 黄金三章爽点分布

### 阶段4：枝叶（13-17步）
13. **正文写作** → 按章纲填充
14. **节奏测试** → 因此/但是检查
15. **自我QA** → 5项必检
16. **迭代修改** → 3次重试
17. **衔接检查** → 确保上下文通顺

## 🔧 核心机制

### 1. 分级执行

| 级别 | 触发 | 处理 |
|------|------|------|
| **日常** | 小修小补 | 直接执行 |
| **普通** | 需要思考 | 保存后执行 |
| **关键** | 大改或新写 | 备份后执行 |

### 2. 爽点嵌入

- **黄金三章**：第1-3章必须有爽点
- **每5章小高潮**：节奏保障
- **每卷大高潮**：章节收束

### 3. 衔接模板

**A型（顺承）**：上一章结果 → 下一章原因
```
因此...
```

**B型（转折）**：上一章结果 → 意外发展
```
但是...
```

**C型（悬念）**：冲突最高点 → 断开
```
就在这时...
```

### 4. QA检查清单（5项必检）

- [ ] 人物目标明确吗？
- [ ] 冲突足够强吗？
- [ ] 节奏快还是拖？
- [ ] 爽点埋了吗？
- [ ] 衔接通顺吗？

### 5. 故事结构诊断（Kimi方法论）

#### 五要素闭环
- 人物：他要什么？为什么现在要？
- 冲突：什么在阻止他？（内在/人际/社会/自然）
- 结构：何时揭示？何时转折？
- 视角：读者知道多少？落差多大？
- 主题：故事问什么无法回答的问题？

#### 三幕检查
- 第一幕：常态→触发→犹豫→新世界
- 第二幕：尝试/失败→中点→崩溃（代价递增）
- 第三幕：灵魂黑夜→最终对决→新常态

#### 节奏测试
- 场景之间是"因此"（因果）还是"但是"（转折）？
- 如果是"然后"（并列），重写

## ⚠️ 铁律

1. **每步必保存** — 完成任何步骤立即保存
2. **错误不过夜** — 出错记录后继续
3. **先诊断再写** — 大章节前先用故事结构诊断
4. **节奏优先** — 写完后必须过"因此/但是"测试
5. **幂等执行** — 同一章节可重复执行
