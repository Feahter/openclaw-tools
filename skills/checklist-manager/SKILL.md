# 清单管理 (Checklist Manager)

将复杂任务拆解为结构化、可执行的清单，确保不遗漏关键步骤。

## 核心原理

**清单的力量**：
- 减少认知负荷
- 防止遗漏关键步骤
- 标准化流程
- 提高执行一致性

## 清单类型

### 1. 执行清单 (DO-CONFIRM)
按记忆执行，然后确认每一步已完成
- 适用于：熟练任务，防止遗漏
- 示例：手术前核查、飞行前检查

### 2. 阅读清单 (READ-DO)
看一步，做一步
- 适用于：复杂或不熟悉的任务
- 示例：设备操作、软件部署

### 3. 沟通清单
确保关键信息传递
- 适用于：团队协作、交接班
- 示例：手术交接、项目交接

### 4. 评估清单
系统化评估标准
- 适用于：决策、评审、诊断
- 示例：代码审查、候选人评估

## 创建方法

### 1. 明确目标
- 这个清单要解决什么问题？
- 谁会在什么场景使用？
- 成功的标准是什么？

### 2. 收集步骤
- 列出所有必要步骤
- 咨询领域专家
- 参考最佳实践

### 3. 精简优化
- 删除非关键步骤
- 合并重复步骤
- 每项控制在5-9个检查点（认知极限）

### 4. 设置触发点
- 在什么情况下使用这个清单？
- 与什么事件绑定？
- 谁来负责执行？

### 5. 测试迭代
- 在实际场景中测试
- 收集反馈
- 持续优化

## 清单设计原则

### 简洁性
- 每项一行，清晰明确
- 使用动词开头
- 避免歧义

### 逻辑性
- 按时间或逻辑顺序排列
- 相关项分组
- 标注关键步骤

### 可操作性
- 可观察、可验证
- 是/否判断，避免模糊
- 明确完成标准

## 示例清单

### 项目启动清单
- [ ] 明确项目目标和范围
- [ ] 识别关键利益相关者
- [ ] 组建核心团队
- [ ] 制定时间计划和里程碑
- [ ] 评估风险和制定应对策略
- [ ] 获得资源批准
- [ ] 召开启动会议

### 写作检查清单
- [ ] 主题明确，一句话能说清
- [ ] 结构清晰，逻辑连贯
- [ ] 开头吸引人
- [ ] 结尾有力量
- [ ] 删除冗余文字
- [ ] 检查错别字和语法
- [ ] 朗读一遍，确保流畅

## 使用方法

1. **输入任务描述** - 描述需要创建清单的任务
2. **选择清单类型** - 根据场景选择合适类型
3. **生成清单** - 获得结构化检查清单
4. **定制优化** - 根据具体需求调整

## 输出格式

生成 Markdown 格式的待办清单：
- 复选框列表格式
- 包含优先级标记（🔴 关键 / 🟡 重要 / 🟢 可选）
- 可直接复制使用

## 运行规则

1. 接收用户输入的任务描述
2. 分析任务类型和复杂度
3. 选择合适的清单类型
4. 生成结构化的检查清单
5. 包含关键检查点和注意事项
6. 输出可直接使用的清单
