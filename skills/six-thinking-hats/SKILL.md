# 六顶思考帽

一次戴一顶，思维更清晰。

> "我从未学会用左手画圆、右手画方。" —— 德·波诺

## 🎯 核心定位

这是一种平行思考方法。传统讨论中，大家同时从不同角度发言，结果是混乱和对抗。六顶思考帽让你一次只从一个角度看问题，然后把所有角度的结论合并。

**核心原理**：
- 传统思考：同时考虑所有因素 → 混乱
- 六顶思考：依次考虑每个因素 → 清晰

---

## 🎩 六顶详解

### ⚪ 白帽子 - 事实

**焦点**：已知什么？需要知道什么？

```
关键词：
数据、信息、事实、证据、记录

核心问题：
├── 我们现在有什么数据？
├── 这些数据可靠吗？
├── 我们缺什么信息？
└── 哪些信息是多余的？

语言标志：
- "数据显示..."
- "据报道..."
- "我们需要核实..."
```

**示例**：
```
⚪ 白帽子分析：
已知：
- DAU较上月下降12%
- 竞品同期增长5%
- 用户调研回收率仅20%

未知：
- 下降是季节性还是持续性？
- 竞品增长是新产品还是营销？
- 用户流失的具体原因？
```

---

### ❤️ 红帽子 - 直觉

**焦点**：感受什么？本能反应是什么？

```
关键词：
感觉、直觉、情绪、预感、情感

核心问题：
├── 我感觉怎么样？
├── 团队的直觉是什么？
├── 这个方案"感觉"对吗？
└── 有没有什么情绪在影响判断？

语言标志：
- "我觉得..."
- "直觉告诉我..."
- "我就是有这个预感..."
```

**示例**：
```
❤️ 红帽子分析：
感受：
- 对这个方案有疑虑，但说不清为什么
- 团队对变革有抵触情绪
- CEO很乐观，但下面的人不踏实

直觉：
- 这个方案过于激进
- 时机可能不对
- 感觉会有人在执行中打折扣
```

---

### ⚫ 黑帽子 - 风险

**焦点**：问题在哪？可能出错什么？

```
关键词：
风险、问题、缺陷、障碍、失败

核心问题：
├── 可能的失败模式是什么？
├── 这个方案有什么漏洞？
├── 什么假设可能不成立？
└── 最坏的情况是什么？

语言标志：
- "问题是..."
- "风险在于..."
- "如果...会怎样"
- "这可能不起作用，因为..."
```

**示例**：
```
⚫ 黑帽子分析：
风险点：
- 技术实现可能比预期复杂
- 团队学习曲线陡峭
- 用户可能不接受新界面
- 竞品可能在我们上线前发布类似功能

假设风险：
- 假设用户愿意学习新操作
- 假设我们有足够的开发资源
- 假设市场环境不会巨变
```

---

### 💛 黄帽子 - 价值

**焦点**：收益在哪？机会是什么？

```
关键词：
收益、利益、价值、机会、好处

核心问题：
├── 最好的结果是什么？
├── 这个方案的价值是什么？
├── 有什么机会可以利用？
└── 成功的话会怎样？

语言标志：
- "好处是..."
- "价值在于..."
- "机会是..."
- "成功的话..."
```

**示例**：
```
💛 黄帽子分析：
价值收益：
- 用户体验大幅提升
- 技术债务减少30%
- 团队技术能力提升
- 可能成为行业标杆

机会：
- 如果第一个推出，可能建立标准
- 如果成功，可以复制到其他产品线
- 技术积累可用于AI产品
```

---

### 💚 绿帽子 - 创意

**焦点**：新想法？不同方案？

```
关键词：
创意、突破、 alternatives、发散

核心问题：
├── 有没有其他方法？
├── 能不能打破常规？
├── 如果不受限制，会怎么做？
└── 激进的想法有什么价值？

语言标志：
- "另一个想法是..."
- "如果...会怎样"
- "这里有一个激进的想法..."
- "跨界灵感是..."
```

**示例**：
```
💚 绿帽子分析：
创意想法：
- 不改现有产品，做一个独立APP试水
- 用AI生成方案，让用户投票决定
- 先在某个小市场做AB测试
- 找竞品合作，而不是竞争

激进想法：
- 完全免费，先圈用户
- 收购竞品而不是自己做
- 用AR/VR重新定义产品体验
```

---

### 🔵 蓝帽子 - 流程

**焦点**：总结什么？下一步做什么？

```
关键词：
总结、决策、计划、流程、控制

核心问题：
├── 我们讨论出了什么？
├── 下一步是什么？
├── 决策标准是什么？
└── 如何保证执行？

语言标志：
- "总结来说..."
- "决定是..."
- "下一步是..."
- "我们需要讨论的是..."
```

**示例**：
```
🔵 蓝帽子分析：
总结：
- 技术可行，有一定风险
- 市场有机会但窗口期有限
- 团队有顾虑，需要管理

决策：
- 批准项目，但分阶段上线
- 第一阶段：3个月内部测试
- 用户反馈好再全面推广

下一步：
- 本周：详细技术方案
- 下周：团队动员
- 下月：第一版上线
```

## 🎯 场景变体

### 创意项目（强化绿帽）

跳过部分黑帽，拉长绿帽时间

```
⚪ 白 → ❤️ 红 → 💛 黄 → 💚 绿(延长) → 🔵 蓝
```

### 风险评估（强化黑帽）

拉长黑帽，仔细审查风险

```
⚪ 白 → ❤️ 红 → ⚫ 黑(延长) → 💛 黄 → 🔵 蓝
```

### 快速决策（极简）

只关注关键帽

```
⚪ 白 → 🔵 蓝（结论先行）
```

### 团队讨论（避免冲突）

让每个人先戴同一顶帽，再切换

```
白 → 红 → 黑 → 黄 → 绿 → 蓝
（所有人同时戴同一顶帽）
```

```markdown

### ⚪ 白帽子
**数据：**
- 当前付费转化率：3.2%
- 竞品A已推订阅，收入增长40%
- 用户调研：60%表示愿意试用
- 服务器成本：$50K/月

**未知：**
- 现有用户对订阅制的接受度
- 竞品A的具体定价策略
- 转型期收入影响

---

### ❤️ 红帽子
**感受：**
- 担心老用户流失
- 对订阅制能否成功没把握
- 团队有抵触——觉得复杂

**直觉：**
- 市场窗口可能正在关闭
- 感觉竞争对手在加速

### ⚫ 黑帽子
**风险：**
1. 现有用户可能因订阅制流失30%
2. 开发订阅系统需要3个月
3. 营销成本可能超支50%
4. 如果失败，声誉受损

**假设可能错：**
- 假设用户愿意为现有功能付费
- 假设我们有足够技术储备

**最坏情况：**
- 用户大量流失 + 开发失败 + 竞争对手超车
```

```markdown
### 💛 黄帽子
**价值：**
1. 稳定的经常性收入
2. 提升产品专注度（只服务付费用户）
3. 可能成为行业标准

**机会：**
1. 竞品转型期正是窗口
2. 订阅数据能指导产品决策
3. 可能建立付费社区

- 12个月订阅收入覆盖服务器成本
- 付费用户增长20%
```

1. 先做免费试用，6个月后再转订阅

- 完全按量付费，没有月费
- 订阅收入与用户分成

- 健身行业的会员模式

- 风险可控，机会大于风险

**决定：**
- 批准，但采用"基础+高级"双轨模式
- 3个月后根据数据决定是否全面切换

**下一步：**

## ⚠️ 使用技巧

### 1. 严格按顺序

不要跳着戴。顺序是有逻辑的：

```
白（事实）→ 红（感受）→ 黑（风险）→ 黄（价值）→ 绿（创意）→ 蓝（决策）

原因：
先看事实 → 感受是建立在事实上的 → 识别风险 → 找价值 → 想办法降低风险捕获价值 → 决定
```

### 2. 红帽时间要短

红帽是宣泄情绪的，不是讨论的。

```
❌ 在红帽阶段长篇大论解释为什么有这个感受
✅ 简短表达感受，然后切到下一顶
```

### 3. 黑帽要建设性

黑帽不是泼冷水，是提前识别问题。

```
❌ "这个方案不行，肯定失败"
✅ "这个方案可能遇到XX问题，我们需要提前准备..."
```

### 4. 绿帽要发散

绿帽阶段不要评判，让想法自由流动。

```
❌ "这个想法不行，因为..."
✅ "还有什么是我们没想过的？"
```

### 5. 蓝帽要有结论

蓝帽阶段必须产出决定和行动。

```
❌ "大家都有道理，继续讨论..."
✅ "综合以上，我们的决定是..."
```

## 🎭 团队使用指南

### 会议前准备

```
1. 提前告知议题
2. 要求参会者准备数据（白帽准备）
3. 指定主持人（蓝帽）
```

### 会议控制

```
1. 每人同时戴同一顶帽
2. 用计时器控制时间
3. 主持人喊"换帽"
4. 禁止跨帽发言
```

### 常见问题

```
Q：有人不按顺序发言怎么办？
A：温和提醒"我们现在在XX帽阶段"

Q：有人坚持自己的观点不听别人说？
A：请他从其他帽子角度重新表达

Q：讨论陷入僵局？
A：切到蓝帽，要求做决定或延期讨论
```
