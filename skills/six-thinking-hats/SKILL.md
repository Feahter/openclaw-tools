# 六顶思考帽 (Six Thinking Hats)

爱德华·德博诺的平行思考方法，让思考更清晰、更全面。

## 核心理念

**平行思考**：同一时间，所有人都戴同一顶帽子，朝同一方向思考。

**分开思考**：将六种思维模式分开使用，而非混在一起。

**全面覆盖**：六顶帽子 = 六个维度，确保没有遗漏。

---

## 六顶帽子

### ⚪ 白帽子 - 客观事实 (White Hat)
**焦点**：信息、事实、数据
**关键问题**：我们知道什么？还需要知道什么？

**使用场景**：
- 收集基础信息
- 确认已知vs未知
- 建立共同的事实基础

**输出示例**：
```
已知：
- 项目预算：50万
- 时间期限：3个月
- 团队规模：5人

未知：
- 第三方API的稳定性
- 用户对新功能的接受度
```

---

### ❤️ 红帽子 - 情感直觉 (Red Hat)
**焦点**：感受、直觉、情绪
**关键问题**：我对此有什么感觉？直觉告诉我什么？

**使用场景**：
- 表达第一印象
- 承认恐惧和担忧
- 捕捉直觉反应

**特点**：
- **不需要理由** - 纯粹表达感受
- **即时反应** - 第一时间的直觉
- **尊重情感** - 情感也是重要信息

**输出示例**：
```
感受：
- 对这个方案感到兴奋
- 担心时间不够
- 直觉认为用户会喜欢
```

---

### ⚫ 黑帽子 - 谨慎批判 (Black Hat)
**焦点**：风险、问题、缺陷
**关键问题**：哪里可能出错？风险是什么？

**使用场景**：
- 风险评估
- 发现潜在问题
- 漏洞分析

**输出示例**：
```
风险：
- 技术风险：新技术团队不熟悉
- 市场风险：竞品可能抢先发布
- 资源风险：关键人员可能离职

缺陷：
- 方案A成本高30%
- 方案B扩展性差
```

---

### 💛 黄帽子 - 乐观价值 (Yellow Hat)
**焦点**：优点、机会、收益
**关键问题**：好处是什么？为什么值得做？

**使用场景**：
- 发现价值点
- 寻找机会
- 建立信心

**输出示例**：
```
价值：
- 提升效率40%
- 节省长期成本
- 增强用户体验

机会：
- 可能打开新市场
- 提升团队技术能力
- 建立行业标杆
```

---

### 💚 绿帽子 - 创意创新 (Green Hat)
**焦点**：创意、可能性、替代方案
**关键问题**：还有什么其他方法？如何跳出框架？

**使用场景**：
- 头脑风暴
- 寻找替代方案
- 突破性思考

**输出示例**：
```
创意：
- 方案C：分阶段发布，降低风险
- 方案D：与合作伙伴联合开发
- 方案E：先用MVP验证，再全面开发

替代思路：
- 是否可以外包非核心部分？
- 是否可以先做POC验证技术可行性？
```

---

### 🔵 蓝帽子 - 控制流程 (Blue Hat)
**焦点**：过程、组织、总结
**关键问题**：我们在做什么？下一步是什么？

**使用场景**：
- 主持讨论
- 控制流程
- 总结决策

**蓝帽子的作用**：
- 决定戴哪顶帽子
- 管理思考时间
- 总结和制定行动计划

**输出示例**：
```
流程控制：
- 已完成白帽子和黑帽子的分析
- 接下来需要黄帽子和绿帽子

行动计划：
1. 继续收集缺失数据（白帽子后续）
2. 制定风险缓解方案（黑帽子后续）
3. 两周后评估进展（蓝帽子回顾）
```

## 使用方法

### 个人使用（30分钟快速版）

**适用场景**：个人决策、快速分析

```
⚪ 白帽子 (5分钟) - 列出所有已知事实
❤️ 红帽子 (3分钟) - 记录直觉感受
⚫ 黑帽子 (7分钟) - 找出所有风险和问题
💛 黄帽子 (5分钟) - 找出所有好处和机会
💚 绿帽子 (7分钟) - 提出创意和替代方案
🔵 蓝帽子 (3分钟) - 总结并制定行动计划
```

### 团队使用（60分钟完整版）

**适用场景**：团队决策、共识建立

```
每顶帽子：10分钟
- 白帽子：共享已知信息，澄清疑问
- 红帽子：每个人表达感受（不解释）
- 黑帽子：集体找出风险和问题
- 黄帽子：集体发现价值和机会
- 绿帽子：头脑风暴替代方案
- 蓝帽子：总结并制定行动计划
```

### 选择性使用

**只关注风险**：
```
⚪ 白帽子 - 收集事实
⚫ 黑帽子 - 深度分析风险
🔵 蓝帽子 - 制定风险缓解计划
```

**只关注创意**：
```
⚪ 白帽子 - 了解约束条件
💚 绿帽子 - 创意发散
💛 黄帽子 - 评估创意的价值
🔵 蓝帽子 - 筛选可行方案
```

## 常见误区

### ❌ 错误做法

| 误区 | 为什么错 | 正确做法 |
|------|---------|---------|
| **同时戴多顶帽子** | 思维混乱 | 一次只戴一顶 |
| **红帽子时给理由** | 破坏直觉 | 只表达感受，不解释 |
| **跳过某顶帽子** | 遗漏维度 | 确保六顶都用到 |
| **黑帽子主导** | 过于悲观 | 平衡使用各帽子 |
| **绿帽子时评判** | 扼杀创意 | 先发散再收敛 |

### ✅ 正确做法

- **明确宣布**："现在戴白帽子"，让所有人知道当前模式
- **时间控制**：每顶帽子最少5分钟，不要匆匆而过
- **全员参与**：每个人都戴同一顶帽子
- **记录成果**：每顶帽子的产出都要记录

## 实际案例

### 案例1：产品功能决策

**背景**：是否开发新功能X？

```
⚪ 白帽子：
- 开发成本：2个月
- 用户调研：60%用户表示需要
- 竞品：已有2家提供类似功能

❤️ 红帽子：
- 感觉时机不对
- 担心资源分散
- 直觉认为应该做

⚫ 黑帽子：
- 可能分散核心功能注意力
- 维护成本增加
- 竞品已有，可能晚了一步

💛 黄帽子：
- 满足60%用户需求
- 可能成为差异化卖点
- 提升用户留存率

💚 绿帽子：
- 方案A：完整功能（2个月）
- 方案B：MVP版本（2周）
- 方案C：与合作伙伴共建

🔵 蓝帽子：
决策：先做MVP版本验证
时间：2周后发布
负责人：产品经理Alice
```


## 进阶技巧

### 顺序调整
标准顺序：白→红→黑→黄→绿→蓝

**其他顺序**：
- **问题优先**：白→黑→黄→绿→红→蓝
- **创意优先**：白→绿→黄→黑→红→蓝
- **快速决策**：白→红→蓝（只关注关键信息）

### 时间分配
- **重大决策**：每顶帽子10-15分钟
- **日常问题**：每顶帽子5分钟
- **紧急决策**：只选2-3顶最关键的帽子

### 团队角色
- **蓝帽子主持人**：控制流程，确保每顶帽子都用到
- **计时员**：控制每顶帽子的时间
- **记录员**：记录每顶帽子的产出

---

## 运行规则

1. **宣布帽子**：明确宣布"现在开始戴X帽子"
2. **专注单一**：一次只使用一顶帽子的思维模式
3. **全员同步**：所有参与者同时使用同一顶帽子
4. **记录产出**：每顶帽子的思考结果都要记录
5. **控制时间**：每顶帽子设定明确的时间限制
6. **最终总结**：用蓝帽子整合所有思考，形成决策
