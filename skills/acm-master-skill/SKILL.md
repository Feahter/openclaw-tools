## 数据结构速查

### 并查集 (Union-Find)

| 维度 | 内容 |
|------|------|
| **用途** | 连通分量、最小生成树、集合合并 |
| **核心优化** | 路径压缩 + 按秩合并 |
| **复杂度** | O(α(n)) ≈ O(1) |
| **代码位置** | `references/algorithms.md` → 并查集 section |

```python
# 工程版核心
class UF:
    def __init__(self, n):
        self.p = list(range(n))
        self.r = [0] * n
    def find(self, x):
        while self.p[x] != x:
            self.p[x] = self.p[self.p[x]]  # 路径压缩（迭代）
            x = self.p[x]
        return x
    def union(self, a, b):
        pa, pb = self.find(a), self.find(b)
        if pa == pb: return False
        if self.r[pa] < self.r[pb]: pa, pb = pb, pa
        self.p[pb] = pa
        if self.r[pa] == self.r[pb]: self.r[pa] += 1
        return True
```

### 线段树 (Segment Tree)

| 维度 | 内容 |
|------|------|
| **用途** | 区间查询、区间修改（适合复杂合并） |
| **核心** | 分治 + 完全二叉树存储 |
| **复杂度** | O(log n) 查询/更新 |
| **代码位置** | `references/algorithms.md` → 线段树 section |

### 树状数组 (Fenwick Tree / BIT)

| 维度 | 内容 |
|------|------|
| **用途** | 单点更新 + 前缀和查询 |
| **核心** | lowbit(i) = i & -i，O(log n) |
| **优势** | 比线段树常数小、实现简单 |
| **代码位置** | `references/algorithms.md` → 树状数组 section |

### LRU 缓存

| 维度 | 内容 |
|------|------|
| **用途** | 最近最少使用淘汰 |
| **核心** | 哈希表 + 双向链表（O(1) 读写） |
| **变种** | LFU（最不经常用）、ARC（自适应） |
| **代码位置** | `references/algorithms.md` → LRU section |

### 堆 (Heap)

| 维度 | 内容 |
|------|------|
| **用途** | 动态 Top-K、优先级队列、最小/最大查找 |
| **核心** | 完全二叉树，父 > 子（max-heap）或父 < 子（min-heap） |
| **复杂度** | O(log n) 插入/删除，O(1) 查顶 |
| **代码位置** | `references/algorithms.md` → 堆 section |

## 算法评价体系

从 4 个维度量化评价（每个算法入库时需要填写）：

| 维度 | 评分 | 说明 |
|------|------|------|
| **性能** | ⭐~⭐⭐⭐⭐⭐ | 时间/空间复杂度的实际表现 |
| **简洁性** | ⭐~⭐⭐⭐⭐⭐ | 实现难度、代码量 |
| **通用性** | ⭐~⭐⭐⭐⭐⭐ | 适用场景广度 |
| **调参难度** | ⭐~⭐⭐⭐⭐⭐ | 参数敏感度（越不敏感越容易用） |

**评分标准**：
- ⭐⭐⭐⭐⭐ = 极佳/无缺点
- ⭐⭐ = 差/明显局限
- 括号内注明具体表现

**入库必需项**：
- [ ] 时间/空间复杂度（含推导）
- [ ] 核心公式或伪代码
- [ ] 竞品对比表
- [ ] 局限性说明
- [ ] 至少 2 个可迁移场景

## 快速参考

**数据结构的 3 个黄金法则**：
1. 查「是否存在」→ 布隆过滤器（内存敏感）或哈希表（精确）
2. 查「Top-K」→ 堆（实时）或快速选择（一次性的）
3. 查「区间和」→ 树状数组（简单）或线段树（复杂合并）

**复杂度的直观感受**（n=10^6）：
| 复杂度 | 实际感受 |
|--------|---------|
| O(1) | 瞬间 |
| O(log n) | ~20 步 |
| O(n) | 10^6 步，可接受 |
| O(n log n) | ~2×10^7 步，边缘 |
| O(n²) | 10^12 步，不可行 |

| 大数越界高精度 | 字符串竖式运算 | O(n²) | references/algorithms.md |
| 递归爆栈改迭代 | 显式栈 / Morris | O(n) | references/algorithms.md |
| 最长回文马拉车 | Manacher | O(n) | references/algorithms.md |
| 有序查找平衡树 | AVL/Treap/FHQ | O(log n) | references/algorithms.md |

**竞赛思维**：
- 遇到区间 → 线段树/前缀和
- 遇到连通 → 并查集
- 遇到回文 → Manacher
- 遇到最值 → 堆/单调栈
- 遇到有序 → 二分
