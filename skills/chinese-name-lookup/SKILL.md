# Chinese Name Fortune Analysis & Recommendation

基于八字喜用神 + 五格剖象法的中文姓名分析与推荐系统。

---

## 工作流程

```
用户输入
    ↓
Step 1: 解析用户信息（姓名/性别/出生时间/出生地）
    ↓
Step 2: 调用八字精算 API 获取喜用神 + 五行能量
    ↓
Step 3: 本地计算五格数理（天格/人格/地格/外格/总格）
    ↓
Step 4: 生成名字推荐方案
    ↓
Step 5: 输出结构化报告
```

---

## Step 1: 解析用户输入

用户可能用多种方式表达，分析并提取：

| 输入类型 | 示例 | 解析 |
|---------|------|------|
| 起名需求 | "帮我宝宝起个名字，姓张，2024年5月1日上午10点出生" | lastname=张, birthdatetime=2024-05-01 10:00 |
| 名字分析 | "姓张名浩然怎么样" | lastname=张, givenname=浩然 |
| 现有名字分析 | "这个名字对八字好吗" | 需要用户提供完整信息 |
| 简化输入 | "张浩然 男 2024-5-1 10:00 北京" | 空格分隔 |

**提取字段**：
- `lastname`: 姓氏
- `givenname`: 名（如果用户提供）
- `gender`: 性别（男/女）
- `birth_year`, `birth_month`, `birth_day`, `birth_hour`: 出生时间
- `birth_province`, `birth_city`: 出生地点
- `parent_lastname`: 父姓（如果母姓+父姓组合起名）

## Step 2: 调用八字精算 API（含 Fallback）

**API 端点**：`https://api.yuanfenju.com/index.php/v1/Bazi/jingsuan`

**配置说明**：将 `scripts/bazi_api.py` 中的 `DEFAULT_API_KEY` 替换为从 doc.yuanfenju.com 申请的 API Key。

**Fallback 机制**：当 API Key 未配置或调用失败时，系统自动切换到 fallback 模式：
- 使用默认喜用神（金、水）继续生成名字
- 在输出中显示 ⚠️ 提示"喜用神为估算值"
- 仍可正常生成五格数理分析

**必填参数**：
| 参数 | 说明 | 示例 |
|------|------|------|
| `api_key` | API 密钥 | `FsF1CsVevk3N17w7oBkSydfSk` |
| `name` | 姓名 | `张三` |
| `sex` | 性别（1=男，0=女）| `1` |
| `type` | 排盘类型（固定=1）| `1` |
| `year` | 出生年 | `2024` |
| `month` | 出生月 | `5` |
| `day` | 出生日 | `1` |
| `hours` | 出生时（24小时制）| `10` |
| `minute` | 出生分 | `0` |
| `zhen` | 真太阳时（固定=1）| `1` |
| `province` | 省份 | `北京市` |
| `city` | 城市 | `北京` |

**调用示例**：
```
GET https://api.yuanfenju.com/index.php/v1/Bazi/jingsuan?api_key=FsF1CsVevk3N17w7oBkSydfSk&name=张三&sex=1&type=1&year=2024&month=5&day=1&hours=10&minute=0&zhen=1&province=北京市&city=北京
```

## Step 3: 本地计算五格数理

### 3.1 笔画数确定

**规则**：
- 单姓：姓笔画数 = 字典笔画数
- 复姓（欧阳、司马等）：姓笔画数 = 第一字笔画 + 第二字笔画
- 名笔画数：取用户指定名或从候选字中查表

**汉字笔画表**：参见 `references/stroke_table.json`

### 3.2 五格计算公式

| 格位 | 计算公式（单姓单名）| 计算公式（复姓）|
|------|-------------------|----------------|
| 天格 | 姓笔画数 + 1 | 复姓第一字 + 第二字笔画 |
| 人格 | 姓笔画数 + 名第一字笔画 | 名第一字笔画（复姓情形复杂，查表）|
| 地格 | 名笔画数 + 1 | 名笔画数 + 1 |
| 外格 | 总格 - 人格 + 1 | 同 |
| 总格 | 姓笔画 + 名笔画数 | 姓总笔画 + 名笔画 |

### 3.3 数理吉凶判定

五格数理有81个数字，每个数字有固定吉凶和含义：

| 类别 | 说明 |
|------|------|
| **吉数** | 1,3,5,6,7,8,11,13,15,16,17,18,21,23,24,25,29,31,32,33,35,37,39,41,45,47,48,52,57,61,63,65,67,68,81 |
| **半吉** | 27,28,30,36,38,39,40,42,43,46,49,50,53,55,58,59,60,62,64,66,69,70,71,72,73,74,76,77,78,79,80 |
| **凶数** | 2,4,9,10,12,14,19,20,22,26,34,43,44,46,49,50,51,54,56,58,59,60,64,66,69,70,71,72,73,74,75,76,77,78,79,80 |

**人格为核心**，最好为吉数，人格为凶数则名字大凶。

### 3.4 三才配置判定

三才 = 天格五行 → 人格五行 → 地格五行

**五行相生相克规则**：
```
相生: 木→火→土→金→水→木
比和: 木木, 火火, 土土, 金金, 水水
相克: 木克土, 土克水, 水克火, 火克金, 金克木
```

**三才配置吉凶**：
- 相生/比和 → 吉
- 相克 → 凶

## Step 4: 名字推荐生成

### 4.1 候选字筛选条件

**必须满足**：
1. **喜用神匹配**：汉字五行属性 ∈ 喜用神列表
2. **五格数理**：人格+地格+总格均为吉数
3. **三才配置**：天格→人格→地格 相生或比和

**优先考虑**：
1. 五格全部为吉数（特别是人格21/23/31等领袖数）
2. 三才配置为相生
3. 字义正向、无歧义谐音
4. 字形平衡（非全繁或全简）

**排除条件**：
1. 家族避讳字（用户指定）
2. 与兄弟姐妹重字（用户指定）
3. 负面历史人物同名
4. 方言谐音歧义（如"杜子腾"等）

### 4.2 候选字五行判断

**方法**：根据字的偏旁部首判断五行：

| 五行 | 偏旁部首 |
|------|---------|
| 木 | 艹, 木, 忄, 讠, 耒 |
| 火 | 灬, 火, 光, 灬 |
| 土 | 阝, 玉, 艸 |
| 金 | 钅, 刂, 戈, 镐 |
| 水 | 氵, 冫, 雨, 冷 |

> 注：这是简化判断，精确五行需要专业字典

### 4.3 评分体系（V2）

**核心原则**：人格和总格为凶数直接排除，三才为凶也排除

**通过条件（必须全部满足）**：
1. 人格数不在凶数列表（FORBIDDEN_NUMBERS）
2. 总格数不在凶数列表（FORBIDDEN_NUMBERS）
3. 三才配置为吉或半吉

**分数构成**：
| 维度 | 得分 | 说明 |
|------|------|------|
| 五格数理 | 50/0 | 人格+总格都为吉则50，否则0 |
| 三才配置 | 25/15/0 | 吉25分，半吉15分，凶0分 |
| 喜用神匹配 | 0-24 | 每匹配一个字+12分 |
| 字义评分 | 10/6 | 全在常用好字表10分，否则6分 |

## 错误处理

| 错误类型 | 处理方式 |
|---------|---------|
| API Key 未配置/失效 | 自动切换 fallback 模式，使用默认喜用神（金、水），输出带⚠️提示 |
| 网络错误 | 重试1次，失败则 fallback |
| 笔画数表中无该字 | 使用偏旁推断五行，默认笔画数8 |
| 候选字不足3个 | 放宽三才要求（允许半吉），直至凑满3个 |
