# Session Insights Skill

## 功能

分析 OpenClaw 的历史 Session，生成结构化复盘报告，包括：
1. 工作概览 — 时间范围、session 数量、主要话题
2. 亮点事件 — 重要成果、新发现、重大决策
3. 效率分析 — 常见任务模式、耗时分布
4. 改进建议 — 基于模式识别的优化方向

## 执行步骤

### Step 1: 获取 Session 列表

执行 `sessions_list` 工具：
- 传入 `activeMinutes: 10080`（最近7天）
- 传入 `messageLimit: 2`（每个 session 2条消息摘要）

这会返回最近 7 天内所有活跃的 session 及其消息摘要。

### Step 2: 分析并归类

根据 session 列表中的摘要和元数据，分析：

**归类维度：**
- 项目/任务类型（代码开发、研究探索、配置调整等）
- 情绪/效率（高产出 vs 探索性）
- 关键决策点
- 遇到的问题及解决方式

**识别模式：**
- 反复出现的任务类型
- 耗时较长的任务
- 多人协作场景（群聊）

### Step 3: 生成报告

按以下结构输出报告（Markdown 格式）：

```markdown
# Session Insights 复盘报告

> 分析范围：最近 7 天
> 生成时间：YYYY-MM-DD HH:mm

## 📊 工作概览

- **活跃 Session 数**: X 个
- **总对话时长**: 约 X 小时
- **活跃日期**: X 天

### 主要工作领域

| 领域 | Session 数 | 占比 |
|------|-----------|------|
| 开发/代码 | X | XX% |
| 配置/运维 | X | XX% |
| 研究/探索 | X | XX% |
| 其他 | X | XX% |

## 🌟 亮点事件

1. **「事件标题」** — Session: [时间]
   - 描述：做了什么
   - 结果：产出什么

（列出 3-5 条最重要的）

## 🔄 任务模式

- 最常见的工作类型：XXXXXXXX
- 平均每次 session 的主要任务：XXXXXXXX
- 协作场景：XXXXXXXX

## 💡 改进建议

1. **建议标题**
   - 现状：XXXX
   - 优化方向：XXXX
   - 预期收益：XXXX

（基于分析结果生成 2-4 条具体可执行的建议）

## 📝 本周总结

（用 2-3 句话总结本周工作重心和状态）

---
*由 Session Insights Skill 自动生成*
```

## 输出要求

- 报告用**中文**输出
- 数据驱动：所有结论必须有 session 历史支撑
- 建议必须具体、可执行，避免泛泛而谈
- 如果历史数据不足（<3 个 session），标注"数据不足，无法生成完整报告"
- 报告中引用具体 session 内容时用「...」标注
