## 二、核心分歧：身强弱判断

### 网站的系统（五行力量法）

| 五行 | 力量 |
|------|------|
| 木 | 60分 |
| **火** | **135分** |
| 土 | 37分 |
| 金 | 51分 |
| 水 | 11分 |
| **合计** | **294分** |

**同类**（生助日主火）：木+火 = 195分  
**异类**（克泄耗日主）：金+土+水 = 99分  
**差值**：195 - 99 = **96分 → 偏旺**

> 这个"同类"=木+火是对的：木生火（木材燃烧生火焰），木火协同增强。
> 辛金（财）被木火消耗，所以金归入异类。

### 引擎 v2 的系统（加权评分法）

| 项目 | 得分 | 权重 |
|------|------|------|
| 月令（卯月=沐浴） | 28分 | 40% |
| 通根（寅中气丙） | 9.8分 | 30% |
| 比劫（丙×1） | 5分 | 20% |
| 印星 | 4分 | 10% |
| **总分** | **46.8/100** | — |

**结论**：46.8分 → 中（临界偏弱）

### ⚠️ 根本分歧

| 系统 | 身强弱 | 差值/得分 |
|------|--------|----------|
| 网站五行力量法 | **偏旺**（差96分） | 大比分优势 |
| 引擎加权评分法 | **中**（46.8分） | 临界偏弱 |

**根因分析**：两套系统的评分逻辑不同，但都"自洽"。关键在于引擎 v2 **缺少五行力量统计表**（木60分、火135分等），因此无法与网站直接对比，也导致 xiyongshen_v2 无法正确综合五行力量与调候/格局的关系。

## 更新 (2026-03-29 16:50) — 喜用神修复完成 ✅

### 修复的Bug

1. **STEM_ELEMENTS key错误** (根本原因)
   - `xiyongshen_v2.py` 中 STEM_ELEMENTS 用整数key {0:0, 1:0...}，但代码用字符串查询
   - 导致所有天干被当作"木"，火日主被误判为木日主
   - 修复：改为正确的字符串key映射 {"甲":"木"...}

2. **五行力量→身强弱映射缺失**
   - `偏旺/旺` → 应视为"强"（需抑），而非"中"
   - 修复：`_effective_strength` 归一化逻辑

3. **偏旺时官鬼水误入喜用**
   - 官鬼水（power=1.5）极弱，且对木旺无益
   - 修复：偏旺时官鬼<15分从喜用移除

4. **fuyi_first时secondary引入弱调候五行**
   - 调候不紧急时，secondary仍被加入喜用
   - 修复：fuyi_first时secondary需power≥15
